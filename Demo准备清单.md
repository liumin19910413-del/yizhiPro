## 当前结论

当前资料已经足够启动第一版 OpenDesign 可点击 Demo。

第一版可默认使用 mock 数据，不需要等待真实接口、真实订单数据或真实前端工程。

## 默认假设

- Demo 只做真实 C 端消费者页面和 B 端商家后台页面。
- Demo 不做开发说明、测试说明、PRD 摘要、状态机图、字段字典和技术架构页。
- 售后流程、状态和操作以 PRD 为最高依据。
- 视觉风格以设计规范为准。
- 参考图用于页面结构、组件样式和真实交互感觉参考。
- mock 订单、售后单、金额、用户、商品、物流和操作日志可由 OpenDesign/Codex 基于 PRD 生成。

## 如果要继续补充，可选资料

这些资料不是启动第一版 Demo 的阻塞项：

- 更贴近真实业务的订单样例。
- 真实商品图片或品牌素材。
- B 端后台现有菜单结构。
- C 端商城现有底部导航或入口结构。
- 售后原因枚举。
- 退款失败原因枚举。
