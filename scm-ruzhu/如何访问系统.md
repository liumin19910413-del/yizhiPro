# 如何访问店商供应链系统

## 问题说明
如果你直接双击打开HTML文件（使用file://协议），会出现404错误或者功能异常。这是因为：
1. 浏览器的安全限制不允许file://协议加载本地资源
2. JavaScript模块和相对路径在file://协议下无法正常工作

## 正确的访问方式

### 方法一：使用已启动的HTTP服务器（推荐）

**服务器已经在运行中！** 直接在浏览器中访问以下地址：

#### 主入口页面
```
http://localhost:8000/index.html
```
这个页面提供了两个系统的入口链接。

#### 直接访问各系统
- **MG管理后台**：http://localhost:8000/mg/index.html
- **Lite商户后台**：http://localhost:8000/lite/index.html

### 方法二：重新启动服务器

如果服务器没有运行，可以通过以下方式启动：

#### 使用启动脚本（推荐）
```bash
cd prototype
./start-server.sh
```

#### 手动启动
```bash
cd prototype
python3 -m http.server 8000
```

### 方法三：使用其他端口

如果8000端口被占用，可以使用其他端口：

```bash
cd prototype
python3 -m http.server 8080
```

然后访问：http://localhost:8080/index.html

## 常见问题

### Q1: 显示"Address already in use"错误
**A:** 说明端口已被占用，可以：
1. 检查是否已有服务器在运行：`lsof -i :8000`
2. 关闭已有服务器：`kill -9 <PID>`
3. 或者使用其他端口

### Q2: 页面显示404错误
**A:** 确保：
1. 服务器正在运行
2. 访问的URL正确（使用http://localhost:8000，不是file://）
3. 在prototype目录下启动服务器

### Q3: 页面显示但功能不正常
**A:** 检查：
1. 浏览器控制台是否有JavaScript错误
2. 是否使用了正确的URL（http://而不是file://）
3. 清除浏览器缓存后重试

### Q4: 如何停止服务器
**A:** 在运行服务器的终端窗口按 `Ctrl+C`
