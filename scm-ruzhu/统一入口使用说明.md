# 统一入口使用说明

## 🎯 功能介绍

统一入口页面（`index.html`）提供了一个便捷的方式，让您可以在同一个浏览器页面中快速切换MG管理后台和Lite商户后台，无需打开多个标签页。

## 🚀 如何使用

### 1. 打开统一入口

```bash
# 方法1：直接打开
open prototype/index.html

# 方法2：使用服务器
cd prototype
./start-server.sh
# 访问 http://localhost:8000/index.html
```

### 2. 选择系统

首次打开会看到欢迎页面，有两种方式进入系统：

**方式1：点击卡片**
- 点击「MG管理后台」卡片进入MG系统
- 点击「Lite商户后台」卡片进入Lite系统

**方式2：点击顶部按钮**
- 点击顶部的「🖥️ MG管理后台」按钮
- 点击顶部的「📱 Lite商户后台」按钮

### 3. 切换系统

进入系统后，可以随时通过顶部按钮切换：

```
┌─────────────────────────────────────────────────────┐
│ 🏪 店商供应链系统  [MG管理后台] [Lite商户后台]  🔄 🗑️ │
└─────────────────────────────────────────────────────┘
```

- 点击「MG管理后台」切换到MG系统
- 点击「Lite商户后台」切换到Lite系统
- 当前激活的系统按钮会高亮显示

### 4. 刷新系统

点击顶部的「🔄 刷新」按钮可以刷新当前系统，相当于重新加载页面。

**使用场景：**
- 在另一个系统中修改了数据，需要刷新当前系统查看
- 系统出现异常，需要重新加载
- 想要查看最新的数据状态

### 5. 重置数据

点击顶部的「🗑️ 重置数据」按钮可以重置所有测试数据。

**注意事项：**
- 会清空所有商户、订单、保证金等数据
- 恢复到初始的测试数据状态
- 重置后需要点击「刷新」按钮才能看到效果
- 操作不可撤销，请谨慎使用

## 💡 使用技巧

### 1. 快速测试联动

**场景：测试Lite端申请 → MG端审核流程**

1. 点击「Lite商户后台」进入Lite系统
2. 提交一个入驻申请
3. 点击顶部「MG管理后台」切换到MG系统
4. 在「商户入驻审批」中看到刚才的申请
5. 审核通过
6. 点击顶部「Lite商户后台」切换回Lite系统
7. 查看申请状态已变为「待签署」

**优势：** 无需打开多个标签页，一个页面完成所有操作

### 2. 对比查看数据

**场景：对比两个系统的数据显示**

1. 在MG系统中查看某个订单的详情
2. 记住订单号
3. 切换到Lite系统
4. 查找相同订单号的订单
5. 对比两边的数据是否一致

### 3. 快速刷新

**场景：在一个系统修改数据后，刷新另一个系统**

1. 在MG系统中发货一个订单
2. 切换到Lite系统
3. 点击「刷新」按钮
4. 查看订单状态是否更新

### 4. 重置测试

**场景：测试完成后重新开始**

1. 点击「重置数据」按钮
2. 确认重置
3. 点击「刷新」按钮
4. 系统恢复到初始状态

## 🔧 技术实现

### iframe技术

使用iframe嵌入两个系统，实现在同一页面中切换：

```javascript
// MG系统iframe
<iframe id="mgFrame" src="mg/index.html"></iframe>

// Lite系统iframe
<iframe id="liteFrame" src="lite/index.html"></iframe>
```

### 切换逻辑

通过显示/隐藏iframe实现快速切换：

```javascript
function switchSystem(system) {
    // 隐藏所有iframe
    frames.forEach(f => f.classList.remove('active'));
    
    // 显示目标iframe
    frames[system].classList.add('active');
}
```

### 数据同步

使用LocalStorage实现数据共享：

```javascript
// 监听storage变化
window.addEventListener('storage', (e) => {
    // 数据变化时自动同步
});
```

## ⚠️ 注意事项

### 1. 浏览器兼容性

推荐使用以下浏览器：
- ✅ Chrome 90+
- ✅ Firefox 88+
- ✅ Safari 14+
- ✅ Edge 90+

### 2. 数据持久化

- 数据存储在浏览器的LocalStorage中
- 清除浏览器缓存会丢失数据
- 不同浏览器的数据不共享

### 3. 刷新说明

- 「刷新」按钮只刷新当前系统
- 如需刷新整个页面，使用浏览器的刷新功能
- 刷新后会保持在当前系统

### 4. 重置数据

- 重置数据会影响两个系统
- 重置后需要手动刷新才能看到效果
- 操作不可撤销

## 🆚 对比传统方式

### 传统方式（多标签页）

```
浏览器标签1: MG管理后台
浏览器标签2: Lite商户后台
```

**缺点：**
- ❌ 需要在多个标签页之间切换
- ❌ 容易混淆当前在哪个系统
- ❌ 占用更多浏览器资源
- ❌ 切换不够流畅

### 统一入口方式

```
浏览器标签1: 统一入口（包含MG和Lite）
```

**优点：**
- ✅ 一个标签页完成所有操作
- ✅ 顶部清晰显示当前系统
- ✅ 节省浏览器资源
- ✅ 切换流畅快速
- ✅ 统一的操作入口
