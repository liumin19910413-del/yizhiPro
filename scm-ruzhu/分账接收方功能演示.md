# 分账接收方功能演示

## 🎯 功能亮点

这个功能解决了一个实际业务痛点：**连锁商户在申请入驻时，品牌总部可能还没有设置分账接收方账户**。

### 传统流程的问题 ❌

```
商户申请入驻 
  ↓
发现没有分账接收方 
  ↓
申请被中断 
  ↓
需要先去添加分账接收方 
  ↓
再重新申请入驻
```

### 新流程的优势 ✅

```
商户申请入驻 
  ↓
发现没有分账接收方 
  ↓
直接在申请流程中添加 
  ↓
一次性提交所有信息 
  ↓
平台一并审核
```

## 💡 业务价值

### 对商户的价值

1. **流程简化**：不需要中断申请去添加分账接收方
2. **一次提交**：所有信息一次性提交，省时省力
3. **体验优化**：智能提示，操作流畅

### 对平台的价值

1. **提高效率**：减少商户重复申请
2. **信息完整**：一次性获取所有必要信息
3. **审核便捷**：所有信息集中展示，便于审核

### 对业务的价值

1. **提高转化率**：减少因流程复杂导致的申请放弃
2. **降低沟通成本**：减少来回沟通的次数
3. **提升满意度**：更好的用户体验

## 📊 数据流转

```
Lite端
  ↓
1. 商户填写入驻申请
2. 商户添加分账接收方（如需要）
3. 提交申请
  ↓
LocalStorage
  ↓
- applications（入驻申请）
  - profitReceiverId: 'pending'
  - receiverRemark: '备注说明'
  - hasPendingReceiverApplication: true
  - pendingReceiverApplicationId: 'PRA001'
  
- profitReceiverApplications（分账接收方申请）
  - id: 'PRA001'
  - applicationId: 'APP001'
  - receiverType: 'company'
  - receiverName: '美丽集团运营部'
  - ... 其他字段
  ↓
MG端
  ↓
1. 审核人员查看申请
2. 看到新申请的分账接收方信息
3. 综合审核
4. 通过或拒绝
```

## 🔧 技术实现

### 核心函数

```javascript
// Lite端
onBrandRatioChange()        // 分成比例变化，加载分账接收方
showAddReceiverModal()       // 显示添加分账接收方弹窗
switchReceiverType()         // 切换账户类型
submitReceiverApplication()  // 提交分账接收方申请
submitApplication()          // 提交入驻申请（包含分账接收方）

// MG端
showAuditModal()            // 显示审核弹窗（增强版）
```

