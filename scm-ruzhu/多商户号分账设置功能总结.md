# 多商户号分账设置功能总结

## 📅 更新时间
2026-01-20

## 🎯 功能概述

在Lite商户端的"入驻管理-申请记录"中，针对"已拒绝"状态的申请，提供了完整的多商户号分账设置功能。该功能支持：
- 选择多个伊智付商户号
- 为每个商户号独立设置分账配置
- 新增完整的分账接收方（对公/对私账户）
- 灵活的分成比例管理

## 🔄 核心功能

### 1. 多商户号选择与管理

#### 界面结构
```
选择伊智付商户号 * (可多选)
☑ YZF202501002 - 美丽集团越秀店
☑ YZF202501004 - 美丽集团总部

商户号管理                    [+ 新增分账接收方]
┌──────────────────────────────────────────────────────┐
│ 伊智付商户号 │ 总部是否分成 │ 分账接收方 │ 分走商户号的收益比例 │ 操作 │
├──────────────────────────────────────────────────────┤
│ YZF202501002 │ [是 ▼]      │ [总部账户▼] │ [10]              │ 删除 │
│ YZF202501004 │ [否 ▼]      │ [-]        │ [0]               │ 删除 │
└──────────────────────────────────────────────────────┘
```

#### 功能特点
- **复选框选择**：顶部显示所有同品牌商户号，支持多选
- **动态表格**：只显示已选中的商户号
- **独立配置**：每个商户号可以独立设置分账参数
- **实时更新**：勾选/取消勾选立即反映在表格中

### 2. 新增分账接收方（完整表单）

#### 2.1 功能入口

- 特点：**独立操作，无需先选择商户号**
- 逻辑：创建接收方 → 生成ID → 在分账设置中选择使用

### 3. 分账设置流程

#### 完整操作流程
```
1. 点击"编辑资料"
   ↓
2. 勾选需要设置的商户号（可多选）
   ↓
3. 商户号出现在表格中
   ↓
4. 如需新增接收方：
   4.1 点击"+ 新增分账接收方"
   4.2 选择对公/对私账户类型
   4.3 填写完整表单信息
   4.4 上传所需证件照片
   4.5 提交创建接收方
   ↓
5. 为每个商户号设置：
   5.1 选择"是否分成"（是/否）
   5.2 选择分账接收方（下拉框）
   5.3 输入分成比例（0-100）
   ↓
6. 验证总分成比例≤100%
   ↓
7. 填写补充说明（可选）
   ↓
8. 提交申请
```

## 📊 数据结构

### 分账接收方（profitReceivers）
```javascript
{
    id: 'PR' + timestamp,           // 分账接收方唯一ID
    receiverName: string,           // 接收方名称
    receiverType: 'company'|'personal', // 账户类型
    contactPhone: string,           // 联系手机号
    
    // 营业执照信息（对公）
    businessLicenseNo: string,      // 营业执照号码
    businessLicenseName: string,    // 营业执照名称
    businessLicensePhoto: string,   // 营业执照照片
    
    // 法人信息
    legalPersonName: string,        // 法人姓名
    legalPersonId: string,          // 法人身份证号
    legalPersonIdFront: string,     // 法人身份证人像面
    legalPersonIdBack: string,      // 法人身份证国徽面
    
    // 收款账户信息
    accountName: string,            // 收款账户名称
    accountNumber: string,          // 收款账户号码
    accountCardNo: string,          // 收款账户卡号
    bankCardPhoto: string,          // 银行卡照片
    
    // 对私账户额外信息
    accountIdNo: string,            // 收款账户证件号
    accountIdFront: string,         // 收款账户证件人像面
    accountIdBack: string,          // 收款账户证件国徽面
    
    remark: string,                 // 备注说明
    status: 'pending',              // 审核状态
    createTime: string              // 创建时间
}
```

### 申请记录的分账设置（profitSettings）
```javascript
profitSettings: [
    {
        merchantId: 'YZF202501002',  // 商户号ID
        receiverId: 'PR001',         // 分账接收方ID（引用）
        ratio: 10                    // 分成比例
    },
    {
        merchantId: 'YZF202501004',
        receiverId: 'PR002',
        ratio: 5
    }
]
```

## ✅ 验证规则

### 商户号选择验证
- ❌ 未选择任何商户号 → 提示"请先在上方选择商户号"
- ✅ 至少选择一个商户号

### 分账设置验证
- ❌ 选择"是"但未选择接收方 → 提示"商户号 XXX 未选择分账接收方"
- ❌ 分成比例≤0或>100 → 提示"商户号 XXX 的分成比例无效"
- ❌ 总分成比例>100% → 提示"所有商户号的总分成比例不能超过100%"
- ✅ 所有验证通过

### 新增接收方验证
- ❌ 未填写必填字段 → 提示"请输入XXX"
- ❌ 未上传必需照片 → 提示"请上传XXX"
- ❌ 文件大小>2M → 提示"文件大小不能超过2M"
- ✅ 所有验证通过 → 创建接收方

## 🔄 业务逻辑

### 分账接收方的独立性
1. **创建阶段**：新增分账接收方 → 生成接收方ID → 保存到profitReceivers表
2. **关联阶段**：在分账设置中选择接收方 → 关联到商户号 → 设置分成比例
3. **复用特性**：一个分账接收方可以被多个商户号选择使用

### 数据流转
```
用户操作
  ↓
勾选商户号 → window.editCurrentProfitSettings 更新
  ↓
设置分账 → 更新对应商户号的配置
  ↓
提交申请 → 构建 profitSettings 数组
  ↓
创建新申请记录 → 状态改为"待审核"
  ↓
关联原申请 → previousAppId 字段
```

## 📝 使用场景

### 场景1：单店商户编辑申请
- 只有一个商户号
- 不需要设置分账（品牌总部为空）
- 直接修改申请人信息后提交

### 场景2：连锁商户编辑申请（单商户号分账）
- 有品牌总部
- 勾选一个商户号
- 设置分账接收方和比例
- 提交申请

### 场景3：连锁商户编辑申请（多商户号分账）
- 有品牌总部
- 勾选多个商户号
- 为每个商户号独立设置分账
- 验证总比例≤100%
- 提交申请

### 场景4：新增分账接收方后使用
- 点击"+ 新增分账接收方"
- 填写完整的对公/对私账户信息
- 上传所需证件照片
- 创建成功后在下拉框中选择
- 设置分成比例并提交

## 💡 注意事项

1. **分账接收方是独立实体**：不依赖于商户号，可以单独创建
2. **先创建后关联**：创建接收方 → 在分账设置中选择 → 关联到商户号
3. **总比例限制**：所有商户号的分成比例总和不能超过100%
4. **文件大小限制**：所有上传的图片文件不能超过2M
5. **必填字段验证**：根据账户类型（对公/对私）验证对应的必填字段
6. **数据兼容性**：新格式完全兼容旧格式的单接收方数据
