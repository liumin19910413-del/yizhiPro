# 商品组合三方分账比例独立设置功能说明

## 更新日期
2026年1月22日

## 功能概述
将"三方分账比例设置"从"编辑商品组合"弹窗中独立出来，作为商品详情面板的一个独立Tab页，方便单独管理和设置。

## 修改内容

### 1. 编辑商品组合弹窗简化

**影响范围：**
- MG管理后台 → 商品组合 → 商品列表 → 编辑

**修改内容：**
- ❌ 去掉"三方分账比例设置"卡片（包括所有输入框和验证逻辑）
- ✅ 添加提示信息："💡 提示：三方分账比例设置已移至商品详情面板的'三方分账比例设置'标签页"
- ✅ 保存商品时不再验证和保存分账比例

**修改函数：**
- `editGoods(goodsId)` - 移除分账比例相关HTML
- `saveEditGoods(goodsId)` - 移除分账比例验证和保存逻辑

---

### 2. 商品详情面板新增Tab

**影响范围：**
- MG管理后台 → 商品组合 → 商品列表 → 点击商品行 → 右侧详情面板

**新增Tab：**
- Tab名称：**三方分账比例设置**
- Tab位置：在"商品价格列表"之后，"组合绑定产品"之前

**Tab顺序：**
1. 商品价格列表
2. **三方分账比例设置** ⭐ 新增
3. 组合绑定产品
4. 不可发货地区
5. 商品邮费设置
6. 商品操作记录
7. 云仓关联信息

---

### 3. 三方分账比例设置Tab内容

#### 3.1 当前分账比例展示

显示三个卡片，分别展示：
- **收款商户号**：蓝色背景，显示当前比例
- **私域运营平台**：浅蓝色背景，显示当前比例
- **供应链平台**：绿色背景，显示当前比例

下方显示总和状态：
- ✓ 当前总和：100%（符合要求）- 绿色提示
- ⚠️ 当前总和：XX%（需要等于100%）- 橙色提示

#### 3.2 修改分账比例表单

**表单字段：**
1. **收款商户号分账比例** *
   - 输入框：数字类型，0-100，支持小数点后2位
   - 单位：%
   - 实时计算总和

2. **私域运营平台分账比例** *
   - 输入框：数字类型，0-100，支持小数点后2位
   - 单位：%
   - 实时计算总和

3. **供应链平台分账比例** *
   - 输入框：数字类型，0-100，支持小数点后2位
   - 单位：%
   - 实时计算总和

**实时验证：**
- 输入时自动计算三方总和
- 总和 = 100%：显示绿色提示"✓ 符合要求"
- 总和 ≠ 100%：显示红色提示"✗ 必须等于100%"

**保存按钮：**
- 点击"保存分账比例"按钮
- 验证总和必须等于100%
- 保存成功后显示提示
- 自动刷新详情面板并停留在当前Tab

---

### 4. 商品价格列表Tab简化

**修改内容：**
- ❌ 去掉"分账比例设置"展示区域
- ✅ 只保留"当前价格"和"历史价格记录"

**原因：**
- 分账比例已独立为单独Tab
- 避免信息重复展示
- 使价格列表更专注于价格信息

## 技术实现

### 新增函数

1. **renderGoodsDetailRatioTab(goods)**
   - 渲染三方分账比例设置Tab内容
   - 显示当前比例和修改表单
   - 返回HTML字符串

2. **updateRatioSum()**
   - 实时计算三方分账比例总和
   - 更新提示信息和样式
   - 验证是否符合100%要求

3. **saveGoodsRatio(goodsId)**
   - 保存商品的三方分账比例
   - 验证总和必须等于100%
   - 记录操作日志
   - 刷新详情面板

### 修改函数

1. **editGoods(goodsId)**
   - 移除三方分账比例设置卡片
   - 添加提示信息

2. **saveEditGoods(goodsId)**
   - 移除分账比例验证逻辑
   - 移除分账比例保存逻辑

3. **showGoodsDetailPanel(goodsId)**
   - 在Tab列表中添加"三方分账比例设置"Tab

4. **switchGoodsDetailTab(tab, goodsId)**
   - 添加'ratio' case，渲染分账比例Tab

5. **renderGoodsDetailPriceTab(goods, prices)**
   - 移除分账比例展示区域

6. **删除 updateEditRatioSum()**
   - 编辑商品弹窗不再需要此函数

## 数据结构

商品数据中的分账比例字段：
```javascript
{
    id: 'GOODS001',
    name: '商品名称',
    merchantRatio: 10,      // 收款商户号分账比例
    operationRatio: 20,     // 私域运营平台分账比例
    platformRatio: 70,      // 供应链平台分账比例
    // ... 其他字段
}
```

默认值：
- merchantRatio: 10%
- operationRatio: 20%
- platformRatio: 70%

## 用户体验优化

### 1. 视觉设计
- 使用不同颜色区分三方：蓝色（收款商户号）、浅蓝色（私域运营平台）、绿色（供应链平台）
- 大字号显示当前比例，突出重要信息
- 实时验证提示使用颜色编码（绿色=成功，红色=错误）

### 2. 交互优化
- 输入时实时计算总和，即时反馈
- 保存成功后自动刷新并停留在当前Tab
- 表单验证失败时给出明确的错误提示

### 3. 信息架构
- 将分账比例设置独立为单独Tab，避免编辑商品弹窗过于复杂
- 价格列表Tab专注于价格信息，分账比例Tab专注于分账设置
- 清晰的Tab导航，方便快速切换

## 注意事项

1. **数据验证**
   - 三方分账比例之和必须严格等于100%
   - 使用 `Math.abs(sum - 100) < 0.01` 进行浮点数比较

2. **操作日志**
   - 每次修改分账比例都会记录操作日志
   - 日志中包含详细的比例信息

3. **浏览器缓存**
   - 测试时记得强制刷新（Cmd + Shift + R）
   - 或在开发者工具中勾选"Disable cache"

4. **数据持久化**
   - 当前使用localStorage存储
   - 实际项目中需要对接后端API
