# 新增分账接收方测试指南

## 测试环境准备

### 浏览器要求
- Chrome/Safari/Firefox 最新版本
- 清除浏览器缓存（Mac: Cmd + Shift + R）

### 访问地址
```
http://localhost:8000/prototype/lite/index.html
```

## 测试场景

### 场景1：初次申请入驻时新增接收方

#### 步骤
1. 登录Lite商户端
2. 点击左侧菜单"入驻管理"
3. 点击"申请入驻"按钮
4. 选择"连锁"类型
5. 勾选一个或多个商户号
6. 点击"下一步"
7. 在分账设置区域，点击"+ 新增分账接收方"按钮

#### 预期结果
- ✅ 弹出"新增分账接收方"弹窗
- ✅ 显示蓝色说明框
- ✅ 默认显示"对公账户"Tab
- ✅ 所有必填字段标记 *

#### 测试点
1. **无需先选择商户号**：直接点击按钮即可打开弹窗
2. **Tab切换**：点击"对私账户"Tab，字段正确切换
3. **文件上传**：
   - 上传小于2M的图片 → 显示"已选择: xxx.jpg"
   - 上传大于2M的图片 → 提示"文件大小不能超过2M"
4. **必填验证**：
   - 不填写名称直接提交 → 提示"请输入分账接收方名称"
   - 不填写手机号 → 提示"请输入联系手机号"
   - 对公账户不上传营业执照 → 提示"请上传营业执照照片"

### 场景2：编辑已拒绝申请时新增接收方

1. 登录Lite商户端

- ✅ 弹出"新增分账接收方"弹窗
- ✅ 无需先勾选商户号复选框

   - 在"分账接收方"下拉框中能看到刚创建的接收方
   - 选择接收方并设置分成比例

## 常见问题

### Q1: 点击"+ 新增分账接收方"没有反应
**解决方案**：
1. 清除浏览器缓存（Mac: Cmd + Shift + R）
2. 检查浏览器控制台是否有JavaScript错误
3. 确认已正确加载 app.js 文件

### Q2: 提示"请先选择商户号"
**原因**：使用了旧版本的代码

**解决方案**：
1. 强制刷新页面（Mac: Cmd + Shift + R）
2. 确认 app.js 文件已更新到最新版本
3. 检查 `showAddEditReceiverModal()` 函数是否已移除商户号检查

### Q3: 文件上传后没有显示文件名
**解决方案**：
1. 检查文件大小是否超过2M
2. 检查 `handleFileUpload()` 函数是否正确执行
3. 查看浏览器控制台是否有错误

### Q4: Tab切换后字段没有变化
**解决方案**：
1. 检查 `switchReceiverTab()` 函数是否正确执行
2. 确认 `company-fields` 和 `personal-fields` 元素ID正确
3. 查看浏览器控制台是否有错误
