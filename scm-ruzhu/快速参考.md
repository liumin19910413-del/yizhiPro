## 🚀 快速启动

```bash

open prototype/index.html
```

- ✅ 一键重置测试数据
- ✅ 自动记忆上次访问

## 📋 核心功能速查

### 商品组合管理

| 功能 | 路径 | 说明 |
|------|------|------|
| 创建商品 | 商品管理 → 商品组合 → + 创建 | 选择供应商和产品组合 |
| 设置价格 | 商品详情 → 价格列表 → + 添加价格 | 配置分成比例和时间 |
| 编辑商品 | 商品列表 → 编辑 | 修改基本信息 |
| 删除商品 | 商品列表 → 删除 | 仅限已下架商品 |
| 上架/下架 | 商品列表 → 上架/下架 | 切换商品状态 |

### 商品价格设置

| 项目 | 说明 | 验证规则 |
|------|------|----------|
| 销售价格 | 商品售价 | 必填，大于0 |
| 折前价 | 商城划线价 | 可选，默认为销售价×1.2 |
| 开始时间 | 价格生效开始 | 必填，必须早于结束时间 |
| 结束时间 | 价格生效结束 | 必填，必须晚于开始时间 |
| 商家比例 | 商家分成% | 必填，0-100 |
| 平台比例 | 供应链平台分成% | 必填，0-100 |
| 运营比例 | 私域运营分成% | 必填，0-100 |
| 总部比例 | 品牌总部分成% | 可选，占商家收益 |

**重要规则：** 商家 + 平台 + 运营 = 100%

### 分成计算公式

```
售价1000元，手续费0.6%：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
手续费 = 1000 × 0.6% = 6元
可分配 = 1000 - 6 = 994元
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
商家10%：994 × 10% = 99.4元
  └─ 总部10%：99.4 × 10% = 9.94元
  └─ 实际：99.4 - 9.94 = 89.46元
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
平台70%：994 × 70% = 695.8元
运营20%：994 × 20% = 198.8元
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
验证：89.46 + 9.94 + 695.8 + 198.8 = 994 ✓
```

## 🎯 常用操作流程

### 创建商品完整流程

```
1. 点击「商品管理」→「商品组合」
2. 点击「+ 创建商品组合」
3. 填写商品名称 ✓
4. 选择分类 ✓
5. 选择供应商 ✓
6. 勾选产品并设置数量
7. 输入销售价格
8. 查看成本预览
9. 点击「创建」
```

### 设置价格完整流程

```
1. 在商品列表点击「详情」
2. 切换到「价格列表」Tab
3. 点击「+ 添加价格」
4. 填写销售价和折前价
5. 设置生效时间段
6. 配置三方分成比例（总和=100%）
7. 设置品牌总部分成（可选）
8. 查看分成金额预览
9. 点击「保存」
```

### 设置地区限制流程

```
方法1：手动设置
1. 商品详情 → 不可发货地区 Tab
2. 点击「设置不可发货地区」
3. 勾选需要限制的地区
4. 点击「保存」

方法2：应用模板
1. 商品详情 → 不可发货地区 Tab
2. 在模板列表中找到合适的模板
3. 点击「应用」
```

## ⚠️ 注意事项

### 商品创建
- ✅ 必须先选择供应商
- ✅ 只能组合同一供应商的产品
- ✅ 至少选择一个产品
- ⚠️ 销售价低于成本价会有警告

### 商品编辑
- ✅ 可以修改名称、分类、价格
- ❌ 不能修改产品组合（需重新创建）
- ⚠️ 修改会记录操作日志

### 商品删除
- ✅ 只能删除已下架的商品
- ⚠️ 删除前会二次确认
- ⚠️ 会同时删除关联的价格数据
- ❌ 删除后不可恢复

### 价格设置
- ✅ 三方分成比例总和必须=100%
- ✅ 结束时间必须>开始时间
- ⚠️ 品牌总部分成占商家收益
- ⚠️ 手续费率固定为0.6%

### 价格终止
- ✅ 只能终止生效中的价格
- ❌ 终止后不可恢复
- ⚠️ 需要重新添加价格

## 🔍 故障排查

### 问题：看不到产品列表
**原因：** 未选择供应商  
**解决：** 先选择供应商，再查看产品

### 问题：分成比例保存失败
**原因：** 三方比例总和不等于100%  
**解决：** 调整比例，确保总和=100%

### 问题：无法删除商品
**原因：** 商品状态为已上架  
**解决：** 先下架商品，再删除

### 问题：价格终止按钮不显示
**原因：** 价格状态不是生效中  
**解决：** 只有生效中的价格才能终止
