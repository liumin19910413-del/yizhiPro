# 2026年1月22日更新总结

## 更新概述

本次更新主要针对Lite商户端的"入驻管理"模块和MG管理后台的"商品组合"模块进行了简化和优化，去除了不必要的复杂流程，并新增了分账接收方管理功能和独立的三方分账比例设置功能。

## 详细更新内容

### 1. 去掉申请编号字段

**影响范围：**
- 入驻管理 - 申请记录列表
- 申请详情弹窗

**修改内容：**
- 列表中不再显示"申请编号"列
- 详情弹窗中不再显示申请编号信息

**原因：**
- 简化界面，减少不必要的信息展示
- 商户号已足够作为唯一标识

---

### 6. 新增分账接收方功能 ⭐

**影响范围：**
- 入驻管理 - 申请记录 - 分账设置

**新增功能：**
- 在"可用分账接收方列表"标题旁边新增"+ 新增分账接收方"按钮
- 点击后弹出"新增分账接收方"表单
- 支持两种账户类型：对公、对私

#### 对公账户表单字段

**必填项：**
- 分账接收方名称
- 联系手机号（11位，1开头）
- 营业执照编号
- 营业执照名称
- 营业执照照片（≤2M）
- 法人姓名
- 法人身份证号（18位）
- 法人身份证照片（人像面+国徽面，各≤2M）
- 收款账户证件照片（人像面+国徽面，各≤2M）

**选填项：**
- 收款账户名称
- 收款账户行号
- 收款账户卡号
- 银行卡照片（≤2M）
- 收款账户证件号

#### 对私账户表单字段

**必填项：**
- 分账接收方名称
- 联系手机号（11位，1开头）
- 收款账户卡号
- 收款账户证件号（18位身份证号）
- 收款账户证件照片（人像面+国徽面，各≤2M）

**选填项：**
- 收款账户姓名

#### 文件上传功能

- 文件大小限制：2M
- 文件类型：仅支持图片格式
- 上传后显示预览
- 可点击"×"删除已上传文件
- 超过大小限制时提示："文件大小不能超过2M"
- 非图片文件时提示："只能上传图片文件"

#### 表单验证

- 所有必填项必须填写
- 手机号格式：1开头的11位数字
- 身份证号格式：18位数字或17位数字+X
- 提交成功后显示："新增分账接收方成功"
- 自动刷新页面，新接收方出现在列表中

#### 数据存储

```javascript
{
    id: 'PR' + timestamp,
    merchantId: brandMerchantId,
    status: 'active',
    receiverType: 'bank',
    accountType: 'company' | 'personal',
    receiverName: '接收方名称',
    contactPhone: '手机号',
    // 对公特有字段
    licenseNumber: '营业执照编号',
    licenseName: '营业执照名称',
    legalName: '法人姓名',
    legalId: '法人身份证号',
    // 共有字段
    accountName: '收款账户名称',
    bankAccount: '银行账号',
    accountId: '收款账户证件号',
    bankName: '银行账户',
    createTime: '创建时间'
}
```

## 注意事项

1. **数据持久化**
   - 当前使用localStorage存储，清除浏览器数据会丢失
   - 实际项目中需要对接后端API

2. **文件上传**
   - 当前为前端模拟，实际项目中需要对接真实的文件上传接口
   - 文件数据存储在DataStore中，刷新页面后会丢失

3. **权限控制**
   - 当前所有用户都可以新增接收方
   - 实际项目中可能需要根据用户权限控制

4. **浏览器缓存**
   - 测试时记得强制刷新（Cmd + Shift + R）清除缓存
   - 或在开发者工具中勾选"Disable cache"
