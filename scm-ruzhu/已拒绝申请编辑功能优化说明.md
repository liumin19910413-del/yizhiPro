# 已拒绝申请编辑功能优化说明

## 📅 修改时间
2026-01-20

## 🎯 功能概述

优化Lite商户端"入驻管理-申请记录"中"已拒绝"状态申请的编辑功能，支持：
- 多商户号分账设置（表格化管理）
- 完整的新增分账接收方表单（对公/对私账户）
- 灵活的分成比例配置
- 优化的操作按钮间距

## 🔄 修改内容

### 1. 多商户号分账设置界面

#### 问题
之前"已拒绝"状态的申请在点击"编辑资料"时，只能设置单个分账接收方，无法支持多个伊智付商户号的分账管理。

#### 解决方案
将"编辑资料"功能的分账设置改为支持多商户号的表格界面：
- 支持选择多个伊智付商户号（复选框）
- 每个商户号可以独立设置是否分成（下拉框：是/否）
- 当选择"是"时，显示分账接收方选择和分成比例输入
- 支持"+ 新增分账接收方"按钮快速添加完整的接收方信息

### 2. 新增分账接收方完整表单

#### 问题
之前只能选择已有的分账接收方，无法在编辑过程中创建新的接收方账户。

#### 解决方案
实现完整的"新增分账接收方"表单，支持对公/对私两种账户类型：

##### 对公账户表单
- 分账接收方名称 *
- 联系手机号 *
- 营业执照号码 *
- 营业执照名称 *
- 营业执照照片 *（不超过2M）
- 法人姓名 *
- 法人身份证号 *
- 法人身份证照片 *（人像面 + 国徽面）
- 收款账户名称 *
- 收款账户号码 *
- 收款账户卡号 *
- 银行卡照片 *（不超过2M）
- 备注说明

##### 对私账户表单
- 分账接收方名称 *
- 联系手机号 *
- 法人姓名 *
- 法人身份证号 *
- 法人身份证照片 *（人像面 + 国徽面）
- 收款账户名称 *
- 收款账户号码 *
- 收款账户卡号 *
- 银行卡照片 *（不超过2M）
- 收款账户证件号 *
- 收款账户证件照片 *（人像面 + 国徽面）
- 备注说明

##### 核心特性
- **独立操作**：无需先选择商户号，直接创建接收方
- **Tab切换**：对公/对私账户类型切换，字段动态显示/隐藏
- **文件上传**：所有图片文件限制2M，超出提示错误
- **完整验证**：根据账户类型验证对应的必填字段
- **即时可用**：创建成功后立即出现在分账接收方下拉框中

#### 3.3 submitEditedApplication 函数
**文件**：`prototype/lite/app.js`

**修改后逻辑**：
1. 从 `window.editCurrentProfitSettings` 读取所有商户号设置
2. 遍历每个商户号，检查是否启用分成
3. 如果启用，验证接收方和比例的有效性
4. 构建 `profitSettings` 数组，包含多个商户号的设置：
   ```javascript
   profitSettings = [
       { merchantId: 'YZF202501002', receiverId: 'PR001', ratio: 10 },
       { merchantId: 'YZF202501004', receiverId: 'PR002', ratio: 5 }
   ]
   ```
5. 计算总分成比例，验证不超过100%
6. 提交新申请

### 4. 操作按钮间距优化

#### 问题
入驻管理-申请记录列表中，操作按钮之间间距太小，显得拥挤。

#### 解决方案
将按钮间距从 `8px` 增加到 `12px`，使界面更加舒适。

**修改位置**：
- "去签署" 和 "详情" 按钮之间
- "编辑资料" 和 "查看原因" 按钮之间
- 主操作按钮和 "分账设置" 按钮之间

## 💡 核心改进点

### 1. 分账接收方的独立性
- **之前**：需要先选择商户号才能新增接收方
- **现在**：接收方是独立实体，可以单独创建，然后在分账设置中选择使用
- **好处**：一个接收方可以被多个商户号复用

### 2. 完整的表单信息
- **之前**：只能填写简单的接收方信息
- **现在**：包含完整的营业执照、法人信息、收款账户、证件照片等
- **好处**：满足实际业务需求，信息完整可审核

### 3. 对公/对私账户支持
- **之前**：只有一种表单类型
- **现在**：支持对公和对私两种账户类型，字段动态切换
- **好处**：适应不同的业务场景

### 4. 文件上传管理
- **之前**：无文件上传功能
- **现在**：支持多个证件照片上传，限制2M大小
- **好处**：完整的资料提交，便于审核
