# 经销商分成编辑功能修复说明

## 问题描述

在MG端的【分润管理】>【经销商单位分成设置】中：
1. 点击"添加经销商"按钮
2. 在弹出的列表中点击某个经销商的"编辑"按钮
3. **问题**：只显示"编辑经销商功能开发中"的提示，无法打开编辑弹窗

## 问题原因

代码中存在两个 `editDistributor()` 函数定义：

```javascript
// 第一个（正确的实现）- line 2321
function editDistributor(distributorId) {
    const distributors = DataStore.get('distributors');
    const distributor = distributors.find(d => d.id === distributorId);
    // ... 完整的编辑功能实现
}

// 第二个（占位符）- line 2523
function editDistributor(id) {
    showToast('编辑经销商功能开发中', 'info');
}
```

由于JavaScript函数重复定义时，后面的会覆盖前面的，导致正确的实现被占位符覆盖。

## 修复方案

删除重复的占位符函数（line 2523-2525），保留完整的编辑功能实现。

## 数据结构

### distributors 数据结构
```javascript
{
    id: 'DIST001',                    // 经销商ID
    name: '广州经销商',                // 经销商名称
    ratio: 10,                        // 经销商分成比例（%）
    tradeCompanyRatio: 90,            // 伊智贸易分成比例（%）
    bindMerchantCount: 3              // 绑定商户数
}
```

### 比例规则
- 经销商分成比例 + 伊智贸易分成比例 = 100%
- 两个比例必须都是数字
- 两个比例必须都在 0-100 之间
- 保存时验证比例合计必须等于 100%

## 技术细节

### 函数列表
1. `showAddDistributorModal()` - 显示经销商列表弹窗
2. `editDistributor(distributorId)` - 编辑单个经销商（已修复）
3. `updateTradeRatio()` - 更新伊智贸易比例（联动）
4. `updateDistributorRatio()` - 更新经销商比例（联动）
5. `updateRatioSum()` - 更新比例合计显示
6. `saveDistributorRatio(event, distributorId)` - 保存经销商比例
7. `batchSetDistributorRatio()` - 批量设置经销商比例
8. `batchSetTradeRatio()` - 批量设置伊智贸易比例
9. `updateDistributorList()` - 更新经销商列表显示

### 事件监听
- 经销商比例输入框：`oninput="updateTradeRatio()"`
- 伊智贸易比例输入框：`oninput="updateDistributorRatio()"`
- 保存按钮：`onsubmit="saveDistributorRatio(event, distributorId)"`

### 数据验证

```javascript
    showToast('两个比例合计必须等于100%', 'error');
    return;
}
```
