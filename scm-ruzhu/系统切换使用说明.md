# 系统切换使用说明

## 快速开始

### 1. 启动服务器

```bash
cd prototype
python3 -m http.server 8000
```

### 2. 访问统一入口

打开浏览器访问：**http://localhost:8000/index-switch.html**

### 3. 使用流程

```
首页（index-switch.html）
    ↓ 点击"MG管理后台"
MG系统（mg/index.html）
    ↓ 点击"返回系统选择"
首页（index-switch.html）
    ↓ 点击"Lite商户后台"
Lite系统（lite/index.html）
    ↓ 点击"返回系统选择"
首页（index-switch.html）
```

## 功能特点

### ✅ 单窗口操作
- 所有操作在同一个浏览器窗口完成
- 无需打开多个标签页
- 类似单页应用的体验

### ✅ 快速切换
- 点击卡片进入系统
- 点击"返回系统选择"按钮返回首页
- 支持浏览器前进/后退按钮

### ✅ 完整功能
- MG系统所有功能正常
- Lite系统所有功能正常
- 数据实时同步

## 常见问题

### Q: 为什么不使用iframe？
A: iframe会导致白屏问题，经过多次尝试无法解决。当前的页面跳转方案更可靠。

### Q: 切换系统会丢失数据吗？
A: 不会。所有数据都存储在localStorage中，切换系统不影响数据。

### Q: 可以同时查看两个系统吗？
A: 如果需要同时查看，可以使用 `index.html`（新标签页方式）。

### Q: 浏览器后退按钮可以用吗？
A: 可以。浏览器的前进/后退按钮完全支持。

## 推荐配置

### 开发环境
```bash
# 使用Python HTTP服务器
cd prototype
python3 -m http.server 8000

# 访问
http://localhost:8000/index-switch.html
```

### 生产环境
```nginx
# Nginx配置示例
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/prototype;
    index index-switch.html;
    
    location / {
        try_files $uri $uri/ =404;
    }
}
```

## 技术细节

### 页面跳转实现
```javascript
// 跳转到系统
function goToSystem(system) {
    window.location.href = system + '/index.html';
}

// 返回首页
onclick="window.location.href='../index-switch.html'"
```
