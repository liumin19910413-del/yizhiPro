# 分成管理页面防闪烁优化说明

## 问题描述

在【分成管理】页面的【品牌总部分成设置】Tab中，页面内容会每5秒闪现一次，影响用户阅读和操作体验。

## 问题原因

系统有一个定时器每5秒自动刷新当前页面，目的是确保数据实时同步。但分成管理页面的数据（分成比例配置）通常不会频繁变化，不需要每5秒刷新，频繁刷新反而导致页面内容闪烁。

## 解决方案

优化定时刷新逻辑，当用户在【分成管理】页面时，跳过自动刷新。

### 修改前
```javascript
setInterval(() => {
    renderPage(currentPage); // 所有页面都刷新
}, 5000);
```

### 修改后
```javascript
setInterval(() => {
    if (currentPage !== 'merchants-profit') {
        renderPage(currentPage); // 跳过分成管理页面
    }
}, 5000);
```

## 技术细节

### 修改的文件
- `prototype/mg/app.js` - `listenStorageChanges()` 函数

### 修改的代码
```javascript
function listenStorageChanges() {
    window.addEventListener('storage', (e) => {
        if (['applications', 'orders', 'deposits', 'products', 'goods'].includes(e.key)) {
            renderPage(currentPage);
            showToast('数据已更新', 'success');
        }
    });
    // 定期刷新数据，确保实时同步
    // 但分成管理页面不需要频繁刷新，避免闪烁
    setInterval(() => {
        if (currentPage !== 'merchants-profit') {
            renderPage(currentPage);
        }
    }, 5000);
}
```

### 为什么这样做有效？
1. **定时器仍然运行**：每5秒执行一次
2. **条件判断**：检查当前页面是否是分成管理页面
3. **跳过刷新**：如果是分成管理页面，跳过renderPage调用
4. **其他页面正常**：其他页面仍然正常刷新

### 数据同步机制
1. **切换页面时刷新**：用户切换到分成管理页面时，会调用renderPage刷新一次
2. **编辑后刷新**：编辑保存后，会调用renderPage刷新
3. **手动刷新**：用户可以通过切换Tab或切换页面来手动刷新
4. **Storage事件**：如果其他标签页修改了数据，会触发storage事件刷新

## 后续优化建议

### 更精细的刷新控制
可以为每个页面配置不同的刷新间隔：
```javascript
const refreshIntervals = {
    'dashboard': 5000,        // 数据概览 - 5秒
    'merchants-audit': 5000,  // 入驻审批 - 5秒
    'merchants-profit': 0,    // 分成管理 - 不刷新
    'orders': 3000,           // 订单管理 - 3秒
    'products': 10000,        // 产品维护 - 10秒
};
```

### 按需刷新
- 只在数据真正变化时刷新
- 使用WebSocket实现实时推送
- 减少轮询，提升性能
