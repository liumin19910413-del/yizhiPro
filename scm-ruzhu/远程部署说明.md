# 远程部署说明

## 当前状态

- **Git仓库**: `git@gitlab.meimeifa.com:yz-product-team/yz-chenjia.git`
- **本地路径**: `/Users/chenjia/yz-chenjia/20260114-SCM`

## 问题分析

远程访问404的可能原因：

1. 远程服务器上没有对应的文件
2. 远程服务器的路径配置不正确
3. 需要手动部署文件到远程服务器

## 解决方案

### 方案一：通过SSH部署（推荐）

如果你有远程服务器的SSH访问权限：

```bash
# 1. 连接到远程服务器
ssh user@chenjia.meimeifa.cn

# 2. 进入网站根目录（需要确认实际路径）
cd /var/www/html  # 或其他路径

# 3. 克隆或更新代码
git clone git@gitlab.meimeifa.com:yz-product-team/yz-chenjia.git
# 或者如果已存在：
cd yz-chenjia
git pull origin master
```

### 方案二：使用rsync同步文件

```bash
# 从本地同步到远程服务器
rsync -avz --delete \
  /Users/chenjia/yz-chenjia/20260114-SCM/ \
  user@chenjia.meimeifa.cn:/var/www/html/20250114-SCM/
```

### 方案三：通过GitLab CI/CD自动部署

创建 `.gitlab-ci.yml` 文件实现自动部署：

```yaml
stages:
  - deploy

deploy_production:
  stage: deploy
  script:
    - rsync -avz --delete ./ user@chenjia.meimeifa.cn:/var/www/html/
  only:
    - master
  tags:
    - deployment
```

### 方案四：使用FTP/SFTP上传

如果只有FTP访问权限，可以使用FTP客户端（如FileZilla）上传文件：

1. 连接到 `chenjia.meimeifa.cn`
2. 找到网站根目录
3. 上传整个 `20260114-SCM` 文件夹

## 临时解决方案

在远程部署配置好之前，可以使用：

### 本地访问（推荐）

```bash
# 启动本地服务器
cd /Users/chenjia/yz-chenjia/20260114-SCM
python3 -m http.server 8000

# 访问地址
http://localhost:8000/prototype/lite/index.html
```

### 使用ngrok进行临时远程访问

```bash
# 1. 安装ngrok（如果还没安装）
brew install ngrok

# 2. 启动本地服务器
python3 -m http.server 8000

# 3. 在另一个终端启动ngrok
ngrok http 8000

# 4. 使用ngrok提供的公网URL访问
# 例如：https://xxxx.ngrok.io/prototype/lite/index.html
```
