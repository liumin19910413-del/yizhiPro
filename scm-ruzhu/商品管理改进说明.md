# Lite端商品管理页面改进说明

## 改进概述

根据您的要求，已将lite端的商品管理页面从**卡片式布局**改为**列表式布局**，以便更清晰地展示商品相关数据。

## 主要改进内容

### 1. 布局方式变更

**原来：卡片式布局**
- 网格布局，每个商品一张卡片
- 视觉效果好，但信息密度低
- 不便于快速对比数据

**现在：列表式布局**
- 表格形式展示，信息密度高
- 便于快速浏览和对比
- 支持更多数据列展示

### 2. 新增数据列

| 列名 | 说明 | 价值 |
|------|------|------|
| 商品封面 | 60x60像素缩略图 | 快速识别商品 |
| SPU商品ID | 商品唯一标识 | 便于查找和管理 |
| 商品名称 | 包含组合产品数量提示 | 了解商品构成 |
| 分类 | 商品分类标签 | 便于分类管理 |
| 销售价格 | 突出显示的售价 | 核心价格信息 |
| 原价 | 划线价格 | 对比优惠幅度 |
| 您的预计收益 | 扣除分成后的实际收益 | 最关心的利润信息 |
| 成本价 | 商品成本 | 了解成本结构 |
| 毛利率 | 利润率百分比 | 盈利能力指标 |
| 更新时间 | 商品信息更新时间 | 数据时效性 |
| 操作 | 查看详情按钮 | 深入了解商品 |

### 3. 功能增强

#### 搜索筛选功能
- **商品名称搜索**：支持模糊匹配
- **SPU ID搜索**：精确查找
- **分类筛选**：按商品分类过滤
- **实时搜索**：输入即时显示结果

#### 商品详情弹窗
点击"详情"按钮可查看：

1. **基本信息**
   - 商品图片、名称、SPU ID
   - 分类、发货方式、更新时间

2. **价格信息**
   - 销售价格、商城原价
   - 成本价格、毛利率

3. **收益分析**
   - 扣除手续费后金额
   - 商家总收益和实际收益
   - 品牌总部分成（如有）

4. **组合产品明细**
   - 产品ID、名称、数量
   - 单价和小计金额

5. **分账明细预览**
   - 各方分账金额和比例
   - 详细的分账说明

### 4. 视觉优化

#### 数据展示
- **收益金额**：绿色高亮显示
- **毛利率**：根据高低使用不同颜色
  - 绿色：>15%（高利润）
  - 橙色：8-15%（中等利润）
  - 红色：<8%（低利润）
- **分类标签**：圆角标签样式

#### 响应式设计
- **桌面端**：完整表格展示
- **移动端**：横向滚动，保持数据完整性
- **最小宽度**：800px确保数据可读性

### 5. 业务逻辑完善

#### 收益计算
```javascript
// 扣除手续费（0.6%）
const netPrice = salePrice * 0.994;

// 商家收益
const merchantProfit = netPrice * (merchantRatio / 100);

// 扣除品牌总部分成（如有）
const actualProfit = merchantProfit * (1 - brandRatio / 100);

// 毛利率
const profitMargin = (actualProfit / salePrice) * 100;
```

#### 分成说明
- 严格按照PRD文档的分成逻辑
- 考虑品牌总部分成的影响
- 提供详细的分账预览

## 使用指南

### 查看商品列表
1. 登录lite端，进入"商品管理"页面
2. 浏览表格中的商品信息
3. 使用搜索框快速查找商品
4. 使用分类下拉框筛选商品

### 查看商品详情
1. 点击商品行的"详情"按钮
2. 在弹窗中查看完整信息
3. 了解收益分析和分账明细
4. 查看组合产品构成

### 搜索和筛选
- **按名称搜索**：输入商品名称关键词
- **按SPU搜索**：输入完整或部分SPU ID
- **按分类筛选**：选择特定商品分类
- **组合使用**：搜索和筛选可以同时使用

## 技术实现

### 核心文件修改
- `prototype/lite/app.js` - 商品管理页面逻辑
- `prototype/shared/styles.css` - 新增样式定义

### 关键函数
- `renderGoods()` - 渲染商品列表
- `filterGoods()` - 搜索筛选功能
- `viewGoodsDetail()` - 商品详情弹窗

### 数据结构
使用现有的商品数据结构，无需修改数据模型。
