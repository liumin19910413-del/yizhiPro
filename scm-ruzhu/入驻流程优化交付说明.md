## 🎯 核心修复

### 问题
连锁申请中的"新增分账接收方"按钮调用了错误的函数，导致添加的分账接收方无法正确关联到品牌总部。

### 解决方案
1. 修改按钮调用：`showAddReceiverModal()` → `showAddReceiverModalForChain()`
2. 新函数会自动获取连锁申请中选中的第一个商户号的品牌总部
3. 确保新增的分账接收方正确关联到品牌总部
4. 新增的接收方会自动出现在所有商户号的下拉列表中

### 代码位置
**文件**: `prototype/lite/app.js`  
**行号**: 2657  
**修改前**: `onclick="showAddReceiverModal()"`  
**修改后**: `onclick="showAddReceiverModalForChain()"`

## 🚀 使用方式

### 访问地址
```
http://localhost:8000/index-switch.html
```

### 测试路径
```
Lite端 → 营销应用 → 店商供应链 → 入驻管理 → 申请入驻
```

### 测试步骤

**单店申请**:
1. 点击"申请入驻"
2. 选择"单店"卡片
3. 填写表单信息
4. 提交申请

**连锁申请（含新增分账接收方）**:
1. 点击"申请入驻"
2. 选择"连锁"卡片
3. 勾选商户号
4. 设置总部分成为"是"
5. 点击"+ 新增分账接收方"
6. 填写对公/对私账户信息
7. 提交分账接收方申请
8. 在下拉列表中选择新增的接收方
9. 提交入驻申请
