# 📊 多优惠计价业务逻辑梳理 v2.0（增加满减满赠）

> 版本：v2.0 | 状态：正式版 | 适用业务：微商城、秒杀、拼团、积分订单 | 最后更新：2026-05-21

> ⚠️ **重要说明**：满减满赠活动仅支持多优惠电商模式，标准模式暂不可用满减满赠活动。

## 一、优惠类型总览

### 1.1 折扣类

| 类型    | 作用对象 | 计算方式     | 备注          |
| :---- | :--- | :------- | :---------- |
| 会员卡折扣 | 所有品  | 原价 × 折扣率 | 改变计价基准，最先执行 |

### 1.2 营销活动类（新增）

| 类型      | 作用对象        | 门槛         | 计算方式            | 分摊方式         | 门槛计算规则              |
| :------ | :---------- | :--------- | :-------------- | :----------- | :------------------ |
| 满减满赠活动  | 指定品类+指定订单类型 | 满X元 or 满X件 | 满足门槛后减固定金额或赠商品/券 | 按适用品当前价比例分摊  | **固定按会员价计算门槛** |

### 1.3 优惠券类（3种）

| 类型    | 作用对象        | 门槛         | 计算方式            | 分摊方式         | 门槛计算规则              |
| :---- | :---------- | :--------- | :-------------- | :----------- | :------------------ |
| 项目抵用券 | 绑定指定品/SKU   | 无门槛，品存在即可用 | 直接从该品扣（全额或固定金额） | 不分摊，直接作用于绑定品 | 无门槛                 |
| 现金券   | 指定品类+指定订单类型 | 满X元 or 满X件 | 满足门槛后减固定金额      | 按适用品当前价比例分摊  | **按该券适用品券前价快照计算门槛** |
| 折扣券   | 指定品类+指定订单类型 | 满X元 or 满X件 | 满足门槛后打折         | 按适用品当前价比例分摊  | **按该券适用品券前价快照计算门槛** |

> **重要说明：券门槛计算规则**\
> 所有优惠券（现金券、折扣券）的门槛判断统一按照「券前价快照」计算。\
> 「券前价快照」不是整单金额，而是每张券独立按自己的适用品范围计算：会员折扣、预定金抵扣、满减满赠活动完成后，且在任意优惠券核销前，冻结该券适用品的金额/件数。

### 1.4 活动类

| 类型          | 性质         | 来源               | 使用限制                       |
| :---------- | :--------- | :--------------- | :------------------------- |
| 预定金         | 活动预付资金     | 用户参与预定金活动时预付     | 仅限预定金活动商品使用，有活动期限，可全额或部分抵扣 |
| **预定金随机立减** | **活动额外优惠** | **预定金活动配置的随机立减** | **仅在预定金活动开启随机立减时生效，最后执行**  |

> **预定金活动说明**

```
- **活动机制**：用户在活动期间预先支付定金，尾款期可享受预定金抵扣优惠
- **适用活动类型**：仅限秒杀活动和拼团活动，普通商品和限时折扣不支持预定金
- **活动配置**：后台可设置哪些具体的秒杀/拼团活动启用预定金功能
- **抵扣方式**： 
    - 全额抵扣：预定金完全抵扣商品价格，商品变为0元（退出后续分摊池） 
    - 部分抵扣：预定金按设定金额抵扣，剩余部分继续参与后续优惠计算

- **计价位置**：在会员折扣之后、满减满赠活动之前执行，确保预定金优先级
- **叠加规则**：与其他优惠可叠加使用，但受商品级「仅限现金支付」限制
- **🎲 随机立减机制**： 
    - **触发条件**：预定金活动开启随机立减功能 + 用户参与该预定金活动
    - **执行时机**：在支付流程中实时计算，基于最终现金支付金额进行随机立减
    - **立减范围**：后台可配置立减金额区间（如1-50元） 
    - **立减对象**：仅对预定金活动商品的最终现金支付金额生效
    - **计算方式**：支付流程中系统随机生成立减金额，从该商品最终现金支付金额中扣除
```

### 1.5 虚拟资产类

| 类型  | 性质     | 来源        | 使用限制                          |
| :-- | :----- | :-------- | :---------------------------- |
| 积分  | 平台虚拟积分 | 消费累积      | 有余额门槛、单笔抵扣上限                  |
| 赠送金 | 平台赠送资产 | 平台赠送/活动奖励 | 有有效期、适用项目限制、订单类型限制、是否叠加会员折扣开关 |

### 1.6 支付类

| 类型       | 性质     | 说明                                                   |
| :------- | :----- | :--------------------------------------------------- |
| 会员卡卡金    | 用户真实储值 | 用户充值的真实资金；可配置适用品类/项目范围；最后消耗；若商品标记为「仅限现金支付」则该商品不可使用卡金 |
| 现金/第三方支付 | 真实资金   | 微信/支付宝/银行卡，兜底支付；亦为「仅限现金支付」商品的唯一可用支付方式                |

## 二（附）、商品级「仅限现金支付」最高优先级配置

### 核心定义

> **「仅限现金支付」= 仅限制支付方式，不限制折扣计价。**  该标记只禁止使用卡金、赠送金、积分、券等非现金支付手段；会员卡折扣属于「改变计价基准」的折扣行为，不受此限制，正常执行。 **预定金随机立减属于活动优惠，不受此限制，正常执行。**

### 品级隔离原则

```plaintext
「仅限现金支付」配置不扩散至整笔订单：
  - 仅影响被标记商品自身的支付方式
  - 同订单内其他未标记商品的优惠计算不受影响
  - 品级隔离，互不干扰
  - 预定金随机立减不受此限制影响
```

### 后台配置项

```plaintext
商品级「仅限现金支付」配置 = {
  开关：是否标记为「仅限现金支付」[ON / OFF]
  └ OFF：该商品正常参与所有优惠计算与支付方式
  └ ON：继续以下约束

  【折扣计价（不受限）】
  STEP1 会员卡折扣：✅ 正常执行（若在折扣适用品范围内）

  【被禁止的支付手段（全部跳过）】
  STEP2 预定金抵扣：❌ 跳过
  STEP2.5 满减满赠活动：❌ 跳过
  STEP3 项目抵用券：❌ 跳过
  STEP4 现金券：    ❌ 跳过
  STEP5 折扣券：    ❌ 跳过
  STEP6 积分抵扣：  ❌ 跳过
  STEP7 赠送金：    ❌ 跳过
  STEP8 卡金：      ❌ 跳过

  【最终结算】
  STEP9 现金支付：支付 STEP1 折后价（或原价，取决于折扣是否适用）
  STEP10 预定金随机立减：🎲 支付流程中实时计算（若该商品参与预定金活动且开启随机立减）

  ⚠️ 备注：该标记在后台商品详情页或批量管理页独立配置
}
```

## 四、完整计价链（核心）

```plaintext
【订单内商品分类】
将订单内所有商品拆分为两个子集：
  子集A：普通商品（无「仅限现金支付」标记）
  子集B：仅限现金支付商品（有标记）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
子集A（普通商品）完整计价链
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP1  会员折扣
  ├ 判断：订单类型是否开启会员折扣 + 商品是否在折扣适用品范围
  ├ 判断：订单是否使用赠送金 + 赠送金与折扣是否配置为「互斥」
  │   → 若互斥且已勾选赠送金：子集A所有商品折扣率强制 = 1（按原价）
  │   → 否则：各品按折扣率正常计算
  └ 输出：各品折后价（= 原价 × 折扣率，或原价）

STEP2  满减满赠活动（新增）
  ├ 获取订单适用的满减满赠活动
  ├ **重要限制：一个订单只能参与一个满减满赠活动**
  │   → 如有多个符合条件的活动，按以下规则选择：
  │   → 优先级1：用户优惠最大的活动
  │   → 优先级2：创建时间最新的活动
  ├ 计算门槛基准：
  │   → 固定基于会员价：sum(各商品STEP1后价格)
  ├ 按阶梯匹配（从高到低）：满足最高阶梯执行
  ├ 减免类型处理：
  │   → 满减：按适用品当前价比例分摊减免额
  │   → 满赠商品：添加赠品到订单，价格=0
  │   → 满赠券：发放券到用户券包
  ├ 叠加规则检查：
  │   → 若配置「不与券共用」：标记券不可用
  │   → 若配置「不与积分共用」：标记积分不可用
  │   → 若配置「不与赠送金共用」：标记赠送金不可用
  └ 更新各品 current_price

STEP3  冻结优惠券门槛快照
  ├ 生成时点：STEP1会员折扣、STEP2满减满赠活动完成后
  ├ 快照粒度：每张现金券/折扣券独立生成一份快照
  ├ 快照内容：该券「适用品范围内」的金额合计、件数合计
  ├ 不计入范围：不在该券适用品范围内的商品、仅限现金支付商品、邮费
  ├ 满减影响：满减满赠先按商品分摊，券快照读取分摊后的商品金额
  └ 目的：确保每张券只按自己的可用品项验门槛，且多张券依次核销时门槛稳定可预期

STEP4  项目抵用券（若有且未被满减满赠活动禁用）
  ├ 每张券直接作用于其绑定品/SKU
  ├ 全额抵用：该品 current_price = 0，退出分摊池
  ├ 固定金额：该品 current_price -= 抵用金额（最低 = 0）
  └ 检查：current_price = 0 的品移出分摊池（但不回刷已冻结的券门槛快照）

STEP5  现金券（若有且未被满减满赠活动禁用，按「满金额门槛从高到低」排序执行）
  ├ **验证门槛（基于该现金券自己的STEP3券前价快照）**
  ├ 按「适用品当前价比例」分摊减免额到各品
  └ 更新各品 current_price

STEP6  折扣券（若有且未被满减满赠活动禁用，按「满金额门槛从高到低」排序执行）
  ├ **验证门槛（基于该折扣券自己的STEP3券前价快照）**
  ├ 按「适用品当前价比例」分摊折扣减免额到各品
  └ 更新各品 current_price

STEP7  积分抵扣（若开启且用户选择使用且未被满减满赠活动禁用）
  ├ 验证：用户积分余额 ≥ 最低余额门槛
  ├ 计算可抵金额：min(用户积分/换算比例, 单笔金额上限, 订单应付额×比例上限)
  ├ 按「分摊池内各品当前价比例」分摊积分减免额
  └ 更新各品 current_price

STEP8  赠送金（若有且用户选择使用且未被满减满赠活动禁用）
  ├ 按赠送金适用品范围过滤（全品/白名单/黑名单）
  ├ 计算可用金额：赠送金余额（不超过适用品当前价总和）
  ├ 按「赠送金适用品当前价比例」分摊减免额
  └ 更新各品 current_price

STEP9  卡金支付（若开启且用户选择使用）
  ├ 按卡金适用品范围过滤（全品/白名单/黑名单）
  ├ 计算可用金额：min(卡金余额, 适用品当前价总和)
  ├ 按「卡金适用品当前价比例」分摊卡金支付额
  └ 更新各品 current_price

**STEP10 现金支付**
  └ 各品 cash_amount = current_price（STEP9后剩余值）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
子集B（仅限现金支付商品）计价链
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP1  会员折扣：✅ 正常执行（若在折扣适用品范围内）
STEP2  满减满赠活动：❌ 跳过
STEP3  优惠券门槛快照：❌ 不生成
STEP4  项目抵用券：    ❌ 跳过
STEP5  现金券：        ❌ 跳过
STEP6  折扣券：        ❌ 跳过
STEP7  积分抵扣：      ❌ 跳过
STEP8  赠送金：        ❌ 跳过
STEP9  卡金支付：      ❌ 跳过
STEP10 现金支付：      支付STEP1折后价

  ⚠️ 备注：子集B商品从子集A的所有分摊池中隔离，
     STEP2~STEP9中各优惠的「适用品总额」均不含子集B商品

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
邮费（独立核算，不进入商品分摊池）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  卡金可支付邮费（按后台开关）
  现金兜底支付邮费（固定可用）
  其他优惠均不可抵扣邮费

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
完整计价链执行顺序总图
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  [原价]
    → STEP1 会员折扣
    → **STEP2 满减满赠活动（新增）**
    → STEP3 按每张券适用品范围冻结门槛快照
    → STEP4 项目抵用券（免单品退池，不回刷门槛）
    → STEP5 现金券（门槛按该券适用品快照计算）
    → STEP6 折扣券（门槛按该券适用品快照计算）
    → STEP7 积分
    → STEP8 赠送金
    → STEP9 卡金
    → STEP10 现金支付
    + 邮费（独立，卡金/现金）
  = 订单最终应付
```

## 五、满减满赠活动详细配置

### 5.1 活动基本配置

```plaintext
满减满赠活动配置 = {
  【基础信息】
  活动名称：活动显示名称
  适用门店：选择门店
  适用类型：微商城订单 / 秒杀订单 / 拼团订单 / 线下订单（多选，至少选一）
  活动时间：开始时间 ~ 结束时间

  【适用范围】
  适用范围：全部可用 / 指定品项可用 / 指定分类可用 / 指定不可用
  自定义套餐：可选；勾选后，线下自定义套餐开单可使用此活动

  【优惠规则】
  优惠规则：阶梯优惠
  门槛计算基准：固定基于会员价；门槛旁提示“按会员价”核算是否适用品项的总金额是否达成门槛条件
  阶梯配置：支持添加下一级优惠，满足最高阶梯执行
    阶梯1：满X元 → 减Y元 / 打Z折 / 无金额优惠 / 选择礼品
    阶梯2：满X元 → 减Y元 / 打Z折 / 无金额优惠 / 选择礼品
    ...

  【参与限制】
  参与限制：全部可参与 / 指定人群参与 / 指定人群不可参与
  人群选择：选择参与人群，支持搜索和多选

  【优惠同享】
  与优惠券优惠共享：默认不勾选
  可与预定金活动共用：默认不勾选
  可与赠送金共用：默认不勾选
  可与积分抵扣共用：默认不勾选
  
  【互斥处理】
  当未勾选对应同享项时：
    - 前端需提示用户选择使用满减满赠活动或其他优惠
    - 系统自动寻优，推荐更划算的方案
    - 用户确认选择后，被排斥的优惠在当前订单中不可用
}
```

- “优惠同享”保持复选框形式，默认均不勾选；本次新增可与预定金活动共用、可与赠送金共用、可与积分抵扣共用。

## 七、设计核心原则总览

```plaintext
【最高优先级规则】
① （最高） 商品级「仅限现金支付」标识高于一切订单级、活动级配置，
   该商品不得参与支付类优惠（卡金/赠送金/积分/券/满减满赠），但可享受会员卡折扣

【计价顺序原则】
② 先折价定基准 → **满减满赠活动** → 冻结券门槛快照 → 再核销优惠 → 最后现金支付
③ 会员折扣、满减满赠先执行，用于生成每张现金券/折扣券自己的券前价快照
④ 现金券、折扣券按各自适用品范围快照校验门槛，后续核销不回刷门槛
⑤ 项目抵用券直接作用于绑定品，影响实付金额，但不改变已冻结的券门槛
⑥ 满金额门槛券先于满件数门槛券；同门槛下按优惠力度从大到小
⑦ 虚拟资产先于真实储值：积分→赠送金→卡金（消耗平台成本资产在前）
⑧ **满减满赠活动在会员折扣后、优惠券前执行，确保门槛计算准确性**

【分摊原则】
⑨ 项目抵用券直接作用于绑定品，不分摊给其他品
⑩ 免单品退出分摊池，不参与后续任何计算
⑪ 「仅限现金支付」品退出支付类分摊池（STEP2~STEP8），但可享受会员折扣
⑫ 每种优惠只在"适用范围内的品"中分摊
⑬ 统一使用最大余数法处理分摊尾差，保证分摊无损
⑭ **满减满赠活动按适用品当前价（会员价）比例分摊，影响后续券门槛快照**

【数据原则】
⑮ 每品保存完整的支付明细快照，退款直接读快照，不重算
⑯ **满减满赠活动仅适用于电商模式，标准模式不支持**
⑰ **一个订单只能参与一个满减满赠活动，多活动冲突时按优惠最大、创建时间排序选择**
⑱ 分账逻辑：仅微信和支付宝等线上支付方式触发分账，卡金不参与分账
⑲ 所有操作写流水记录，支持审计和对账
⑳ 退款按品级快照原路逆向，不影响其他品
㉑ **满减满赠活动的减免金额一旦生成不可退回，满赠商品可退货但需退回库存**
```
