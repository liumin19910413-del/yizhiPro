# AI运营大脑界面设计规范

本文档基于当前四张参考截图提炼，用于后续登录页、运营驾驶舱、私域列表与客群分层页面的一致性还原。

## 1. 设计定位

AI运营大脑的界面应保持“美业 SaaS 后台 + AI 辅助经营”的气质：信息密度高、结构清晰、色彩轻盈，避免营销页式的大面积装饰。页面首要目标是让门店经营者快速判断状态、进入任务、处理客户。

关键词：轻量、可信、清爽、可扫描、数据驱动、AI 辅助但不喧宾夺主。

## 2. 色彩规范

### 2.1 品牌与功能色

| 用途 | 色值 | 使用场景 |

| 深品牌蓝 | `#253f83` | 登录插画主标题、重要标题 |
| 粉色强调 | `#ff6f9a` | 私域模块选中态、美业特色操作入口、警示性标签 |
| 成功绿 | `#35c28d` | 正向状态、活跃客、可用数据 |
| 警示橙 | `#f6a94d` | 低于行业值、待处理、机会提醒 |
| 紫色辅助 | `#8d6df5` | 搜索、AI、轻量分群 |

## 3. 字体与字号

默认字体栈：

```css
font-family: "PingFang SC", "Microsoft YaHei", "Helvetica Neue", Arial, sans-serif;
```

| 场景 | 字号 | 字重 | 行高 |
| --- | --- | --- | --- |
| 登录页大标题 | 44-48px | 700 | 1.25 |
| 页面标题 | 18-20px | 600 | 1.4 |
| 模块标题 | 15-16px | 600 | 1.4 |
| 表单/表格正文 | 13-14px | 400 | 1.6 |
| 辅助说明 | 12px | 400 | 1.5 |
| 数据大数字 | 22-28px | 600-700 | 1.2 |

字距保持默认 `0`，不使用负字距。紧凑表格允许减少行高，但不要压缩到影响中文阅读。

## 4. 布局规范

### 4.1 后台主框架

- 后台主框架优先对齐线上应用中心截图：左侧单栏导航宽约 `222px`，白底、弱分割线、紫色品牌态，不再使用双层侧栏作为默认首页结构。
- 内容区使用白色页面底，顶部为页签导航，当前页签使用紫色文字和底部短横线；页签下方使用浅灰分割带形成线上后台的层次感。
- 首页以“营销应用”应用中心为基准：按 `开单 / 客情 / 公域` 分组展示功能模块，分组标题左侧使用轻量图标。
- 应用模块卡片采用浅色底纹、细边框、小圆角和左侧彩色图标，不使用重阴影、大面积蓝色看板或营销式装饰。
- 初始化弹窗仍覆盖在后台页面上方，弹窗结构和视觉独立于后台应用中心，不随后台页面改版而变化。

### 4.2 登录页

- 画布参考尺寸：`1373 × 744`。
- 顶部 Logo 距左约 `20px`，距上约 `20px`。
- 中心登录容器宽约 `1176px`，高约 `556px`，距上约 `97px`。
- 容器左侧为插画区，宽约 `728px`；右侧表单区宽约 `448px`。
- 登录表单左边距约 `60px`，输入框宽 `336px`，高度 `56px`，按钮高度 `56px`。
- 右上角二维码入口用斜切角呈现，不作为主视觉抢占表单权重。
- 右侧浮动工具条宽约 `74px`，固定在画面右侧中部。

## 5. 组件规范

### 5.1 卡片

- 圆角：`6-8px`。
- 边框：`1px solid #dfe7f2`。
- 背景以白色或极浅色为主，避免强投影。
- 卡片内标题与内容保持 `8-12px` 间距，数据数字需明显高于辅助说明。

### 5.2 输入框

- 高度：登录页 `56px`，后台筛选区 `32-36px`。
- 边框：`#d7d7d7`，聚焦态改为 `#2f7cf6`。
- 占位文字：`#c3c7ce`。
- 图标放在左侧，图标区与文本之间使用浅分割线。

### 5.3 按钮

- 主按钮：品牌蓝底、白字、圆角 `3-4px`，高度 `40-56px`。
- 后台轻量按钮：白底蓝边或浅蓝底，避免过度饱和。
- 危险或提醒类操作使用粉色，不与主蓝按钮竞争。

### 5.4 表格

- 表头背景：`#f6f6f8`。
- 行高：`64-100px`，根据内容密度调整。
- 标签使用浅底色，粉色标签用于肤质/问题关键词，蓝色标签用于 AI 类型。
- 重要待处理行可用极浅粉背景 `#fff2f5` 提示。

### 5.5 运营数据卡

- 数据指标采用“数字 + 占比/环比 + 简短定义”的结构。
- 活跃、流失、无记录等状态使用稳定色彩映射，饼图与卡片色保持一致。
- 分群卡片以浅色底承载，按重要性使用蓝、橙、紫、绿等不同轻色系，避免整页单一蓝色。

## 6. 图形与插画风格

- 插画使用柔和 3D/半透明质感，蓝紫为主，辅以粉色美业人物元素。
- 背景可使用浅网格增强科技感，但网格线需低对比。
- 插画不承担操作功能，需服务登录页“智慧门店经营”的心智。
- AI相关图形可使用圆环、连接线、芯片、数据卡、气泡等元素，但保持轻盈。

## 7. 交互状态

- Hover：按钮轻微加深或上浮 `1px`，后台卡片仅改变边框色。
- Focus：输入框使用蓝色描边和弱阴影。
- Disabled：文字 `#b4bdca`，背景 `#f5f7fa`。
- Loading：优先使用局部加载，不遮盖整个后台。

## 8. 文案风格

- 登录页文案短促直接，例如“密码登录”“请输入您的登录账号”“请输入密码”。
- 后台说明文案使用业务动作表达，不使用抽象营销词堆叠。
- AI洞察应先给结论，再给依据和建议动作。

## 9. 本次登录页还原清单

- 品牌 Logo：左上角“伊智科技 / MEIMEIFA.COM”。
- 中央白色登录面板：左图右表单，右上二维码折角。
- 插画主文案：“让门店增长更轻松”“智慧门店经营”。
- 表单：账号、密码、记住密码、忘记密码、登录按钮。
- 底部下载入口：商家版下载、大掌柜下载、客户端下载。
- 右侧浮动入口：移动收银、官网首页、咨询建议、申请试用。

## 10. 登录后业务初始化规范

- 默认演示账号进入系统后，如账号未初始化，需自动弹出“业务初始化”弹窗。
- 弹窗采用左右向导结构：左侧为“初始化步骤”列表，右侧为当前步骤的内容确认区。
- 左侧步骤列表使用白色卡片、浅灰未完成态、品牌蓝当前态、浅绿完成态，数字圆点与截图风格一致。
- 右侧内容区顶部显示 `Step X / N`、当前步骤标题、说明文案和“AI已生成内容，等待确认”标签。
- 初始化步骤需按页面性质适配交互：用户决策型步骤（如选择行业）不展示进度条；系统自动执行型步骤才展示本步进度条。
- 系统自动执行型步骤进度达到 100% 前，主按钮不可点击，不能进入下一步；进度完成后主按钮自然高亮。
- 用户决策型步骤要做成一体化选择面板，不使用单选圈式功能控件，不强行展示“初始化进度”。
- 当步骤内容本身包含子项进度（如品项策略分组）时，不再额外展示顶部总进度条，进度应融入每个子项卡片或分组模块内部。
- 初始化向导不再单独展示“组织架构初始化”步骤，组织角色和员工档案可作为后台行业语境数据保留，但不占用商家的初始化确认链路。
- 初始化过程不提供“跳过本步”按钮，避免商家感知初始化链路不完整。
- 顶部说明文案保持克制，只保留一句必要描述；较长 AI 说明默认收进“AI说明”折叠项，需要时再展开。
- 当前步骤内容以轻量生成态为主；除行业选择外，不出现策略选择按钮、人工配置盒或强配置感表单。
- 底部固定操作区仅包含“上一步”和主按钮；清洗用户记忆完成后显示“一切准备就绪，开始让AI运营大脑帮您打工吧！”。
- 初始化步骤固定为：选择行业、品项初始化、会员数据初始化、清洗用户记忆。
- 第一步“选择行业”支持多选，行业组合会影响后续品项、套餐、会员标签和AI策略初始化。
- 第一步选择行业后，后续所有初始化内容必须根据当前行业组合动态生成，包括服务品项、套餐模板、会员生命周期、消费流样本、用户记忆来源与清洗标签，不允许固定使用单一行业数据。
- 行业组合需要随初始化状态持久化；进入 `AI私域大脑` 后，首屏搜索示例、AI洞察、最近会员、热销品项也要延续所选行业组合语境。
- “品项初始化”需要强化商家体验感，应按品项旅程展示 `引流品 / 体验品 / 升单品 / 复购品 / 高价值品 / 利润品` 六类结构，让商家感知系统是在按行业经营逻辑初始化，而不是普通导入项目。
- 六类品项采用精致卡片手风琴清单交互：类目头展示名称、经营说明、品项数量、胶囊式初始化进度和生成状态；默认全部收起，用户点击类目后才展开具体项目，且允许多个类目同时展开。
- 品项手风琴必须在右侧弹窗内容框内独立滚动，展开多个类目时不能挤压或遮挡底部“上一步 / 下一步”按钮。
- 展开类目需以表格化方式展示具体品项名和售价，并可补充行业分类、生成原因；未完成项使用弱化或等待态。
- “会员数据初始化”需要先展示企微客服号二维码授权，引导商家扫码绑定门店客服号，文案聚焦“授权客服号，先把客户关系建起来”。
- 授权页需要解释清楚当前步骤边界，并用路径化表达替代平铺功能卡：同步客户关系、自动会员建档、模拟消费周期。当前步骤只同步客服号通讯录并生成会员档案，再模拟会员生命周期消费数据；不要暗示此时已经拉取聊天记录和朋友圈。
- 真实业务中应是扫码授权成功后自动进入初始化进度；demo 中不展示额外确认按钮，进入该页面约 3 秒后自动模拟“扫码授权成功，开始初始化数据”，再启动下方进度。
- 授权页必须降低商家的不放心感，使用轻量安全说明而非长篇协议，例如“仅用于本次 AI 运营体验”“当前仅同步通讯录建档”“后续授权后可接入聊天和朋友圈触点”。安全说明宜做成一条浅绿提示条，不要做成过重的功能卡。
- 企微二维码只作为授权前动作卡，不长期固化在右侧；授权成功后短暂停留成功态，然后授权卡整体收起，初始化进度面板成为页面主体，保证一屏尽量完整展示。
- 授权完成前不展示通讯录同步、会员建档、消费记录生成等进度内容，避免页面过早复杂化；扫码授权完成后才开始出现生成进度条和结果数字。
- 授权后的会员数据初始化只保留一组轻量生成反馈：已同步通讯录人数、已识别微信客户数、建档成功会员数、模拟消费记录条数，以及当前生成进度。进度达到 100% 后，底部主按钮允许直接进入“清洗用户记忆”，不再额外展示会员生命周期结果页。
- “清洗用户记忆”不随行业变化，固定展示 8 个待清洗数据来源：会员资料、消费订单、卡项 / 套餐记录、小程序行为、微信聊天、朋友圈动态、服务记录、员工回访记录。
- 用户记忆清洗页需要逐项清洗：每个来源独立展示“待清洗 / 清洗中 / 已完成”、已清洗条目数和单项进度动效；标签必须站在老板能理解的运营机会维度表达，例如高价值客户、到期复购、沉睡预警、升单机会、预约意向、风险客户、加项推荐、回访任务，而不是技术字段名。必须 8 项全部清洗完成后，底部主按钮才可显示“一切准备就绪，开始让AI运营大脑帮您打工吧！”。
- 用户记忆清洗页标题下保留一句简洁说明文案，不单独展示“8/8 已完成”计数条；8 个数据源卡片就是页面主体，卡片高度保持紧凑，避免下方空白过大。
- 除行业选择外，初始化过程不展示人工选择项或策略按钮；品项、会员消费、用户记忆均默认由 AI 自动创建和清洗。
- 初始化向导应尽量清爽丝滑：减少配置感，只保留当前系统正在生成的内容、进度和结果。
- 最后一步为“清洗用户记忆”，点击“一切准备就绪，开始让AI运营大脑帮您打工吧！”后关闭弹窗，并自动进入 `私域 > AI私域大脑`。

## 11. 后台菜单与命名规范

- 一级主菜单固定为：预约、开单、私域、智能体、营销、报表、设置。
- 一级菜单保持白底低饱和图标，当前选中项使用紫色浅底和左侧短竖条，避免使用大块粉色选中态。
- 私域下二级菜单中，原“用户记忆”统一命名为“AI私域大脑”。
- 后台默认首屏按线上“营销应用”应用中心样式展示，模块包括秒杀、预定金、积分营销、发券宝、安心付系列、会员等级、预约、企业微信、满意度评价、短信营销、活动意向调研、AI 创作神器等。
- 后台页面的卡片、按钮、表格、搜索框必须继续复用本文档定义的色彩、圆角、间距和交互状态。
