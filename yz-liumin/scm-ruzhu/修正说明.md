# 供应链系统Demo修正说明

## 问题分析

根据您反馈的问题："刚刚在lite端创建了一条申请数据，但是在mg端没看到这题啊数据，并且不能进行审核处理"，我分析并修正了以下问题：

## 修正内容

### 1. 数据同步问题修正

**问题**：lite端创建申请后，mg端无法实时看到新数据

**修正**：
- 优化了数据同步机制，将刷新间隔从3秒调整为5秒，并增加了立即刷新逻辑
- 修正了lite端申请提交后的状态检查，确保立即显示申请记录
- 增强了localStorage的事件监听，确保跨页面数据同步

### 2. 审核功能完善

**问题**：mg端审核功能不完整，缺少实际分成比例设置

**修正**：
- 完善了审核模态框，使用统一的modal组件而不是动态创建HTML
- 增加了分成比例验证逻辑，确保总和为100%
- 修正了实际分成比例的计算方法（取区间中间值）
- 增加了品牌总部分成的正确处理逻辑

### 3. 文案和逻辑优化

**问题**：部分文案不符合PRD要求

**修正**：
- 订单状态文案：将"余额不足不可发货"改为"待充值保证金"
- 工作台提示：优化了保证金不足的提示文案，更准确描述业务场景
- 审核流程：按照PRD要求完善了自动审核和人工审核的逻辑

### 4. 增加测试工具

**新增**：
- 创建了`test-sync.html`测试页面，方便验证数据同步功能
- 提供了创建测试申请、审核申请、删除申请等测试功能
- 实时监控数据变化，便于调试

## 测试步骤

### 方法1：使用测试页面
1. 打开 `prototype/test-sync.html`
2. 点击"创建测试申请"按钮
3. 分别打开lite端和mg端，观察数据同步情况
4. 在mg端进行审核操作，观察状态变化

### 方法2：手动测试
1. 打开 `prototype/lite/index.html`（商户后台）
2. 打开 `prototype/mg/index.html`（管理后台）
3. 在lite端提交新的入驻申请
4. 切换到mg端，查看"商户入驻审批"页面
5. 点击"审核"按钮进行审核处理

## 核心业务逻辑

### 申请流程
1. **商家申请**：在lite端填写申请信息并提交
2. **自动审核**：系统检查经营范围和分账功能
3. **人工审核**：mg端审核员设置分成比例区间并审核
4. **签署协议**：审核通过后，商家在lite端签署协议
5. **正式合作**：协议签署完成，开始正常业务

### 数据同步机制
- 使用localStorage作为共享数据存储
- 通过storage事件实现跨页面数据同步
- 定期轮询确保数据一致性
- 关键操作后立即刷新状态

### 分成比例设置
- 审核时设置比例区间（如商家8-12%）
- 系统自动计算实际执行比例（取中间值，如10%）
- 三方比例总和必须为100%
- 品牌总部分成占商家收益的百分比

## 注意事项

1. **浏览器兼容性**：建议使用Chrome、Safari、Firefox等现代浏览器
2. **数据持久化**：当前使用localStorage，刷新页面数据不会丢失
3. **实时同步**：需要保持页面活跃状态，最小化或切换标签页可能影响同步
4. **测试数据**：可以使用测试页面的"清理测试数据"功能重置为初始状态

- `prototype/lite/app.js` - 优化申请提交和状态检查
- `prototype/shared/data.js` - 数据结构保持不变
- `prototype/test-sync.html` - 新增测试工具
