# 页面编辑器闪退问题修复说明

## 问题描述

**问题现象**：
- 点击"创建页面"按钮进入页面编辑器后
- 过了一会儿（约5秒）会自动闪退
- 自动返回到商城装修的内容管理页面

**影响范围**：
- 无法正常使用页面编辑器
- 无法添加和配置组件
- 无法保存页面

## 问题原因

### 根本原因
系统中有一个定时器每5秒自动刷新页面，用于保持数据实时同步：

```javascript
setInterval(() => {
    const excludePages = ['merchants', 'merchants-profit', 'goods', 'products', 'mall-goods'];
    if (!excludePages.includes(currentPage)) {
        renderPage(currentPage);
    }
}, 5000);
```

### 问题分析
1. 当用户进入页面编辑器时，`currentPage`仍然是`'mall-decoration'`
2. 页面编辑器不在排除列表中
3. 5秒后定时器触发，调用`renderPage('mall-decoration')`
4. 重新渲染商城装修页面，导致编辑器被关闭
5. 用户看到的效果就是"闪退"

## 解决方案

### 修复思路
添加一个全局标记`isInPageEditor`来标识是否在页面编辑器中，在定时器中检查这个标记，如果在编辑器中就不执行自动刷新。

### 代码修改

#### 1. 添加全局标记
```javascript
let currentPage = 'dashboard';
let currentProfitTab = 'brand';
let currentMerchantsTab = 'audit';
let isInPageEditor = false; // 新增：标记是否在页面编辑器中
```

#### 2. 修改定时器逻辑
```javascript
function listenStorageChanges() {
    window.addEventListener('storage', (e) => {
        if (['applications', 'orders', 'deposits', 'products', 'goods'].includes(e.key)) {
            // 如果在页面编辑器中，不自动刷新
            if (!isInPageEditor) {
                renderPage(currentPage);
                showToast('数据已更新', 'success');
            }
        }
    });
    
    setInterval(() => {
        const excludePages = ['merchants', 'merchants-profit', 'goods', 'products', 'mall-goods'];
        // 如果在页面编辑器中，不自动刷新
        if (!excludePages.includes(currentPage) && !isInPageEditor) {
            renderPage(currentPage);
        }
    }, 5000);
}
```

#### 3. 进入编辑器时设置标记
```javascript
function showPageEditor(pageId) {
    const pageData = JSON.parse(sessionStorage.getItem('editingPage'));
    const isNewPage = sessionStorage.getItem('isNewPage') === 'true';
    
    // 设置标记，防止自动刷新
    isInPageEditor = true;
    
    const content = document.getElementById('page-content');
    content.innerHTML = renderPageEditor(pageData, isNewPage);
    
    initPageEditor(pageData);
}
```

#### 4. 退出编辑器时清除标记
```javascript
function closePageEditor() {
    if (confirm('确定要关闭编辑器吗？未保存的更改将丢失。')) {
        sessionStorage.removeItem('editingPage');
        sessionStorage.removeItem('isNewPage');
        // 清除标记，恢复自动刷新
        isInPageEditor = false;
        renderPage('mall-decoration');
    }
}
```

#### 5. 保存页面时清除标记
```javascript
function savePage(status) {
    // ... 保存逻辑 ...
    
    // 清除临时数据
    sessionStorage.removeItem('editingPage');
    sessionStorage.removeItem('isNewPage');
    
    // 清除标记，恢复自动刷新
    isInPageEditor = false;
    
    // 返回列表
    renderPage('mall-decoration');
}
```

## 技术说明

### 为什么需要定时刷新
- 保持数据实时同步
- 多用户协作时及时更新数据
- 避免数据不一致

### 为什么编辑器不能刷新
- 编辑器是全屏覆盖的独立界面
- 刷新会导致编辑器被销毁
- 用户正在编辑的数据会丢失
- 影响用户体验

### 解决方案的优点
- ✅ 简单有效
- ✅ 不影响其他页面的刷新
- ✅ 退出编辑器后自动恢复刷新
- ✅ 代码改动最小
- ✅ 易于维护

## 注意事项

### 开发注意
1. 如果新增其他全屏编辑器，也需要设置类似的标记
2. 确保进入和退出编辑器时都正确设置标记
3. 保存、取消、关闭等所有退出路径都要清除标记

### 测试注意
1. 测试时需要等待足够长的时间（至少10秒）
2. 测试各种退出编辑器的路径
3. 确认退出后刷新功能恢复正常
