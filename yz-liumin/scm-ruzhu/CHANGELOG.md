## [1.7.8] - 2026-01-15

### 🐛 Bug修复：分成管理子Tab点击无效问题

#### 问题描述
在【商户入驻】>【分成管理】Tab中，点击【品牌总部分成设置】和【经销商单位分成设置】两个子Tab无效，无法切换。

#### 问题原因
- `switchProfitTab`函数使用了`document.querySelectorAll('.profit-tab')`
- 这会选择页面上所有的`.profit-tab`元素
- 包括主Tab（入驻审批、分成管理）和子Tab（品牌总部、经销商单位）
- 导致选择器冲突，子Tab无法正常切换

#### 解决方案
修改`switchProfitTab`和`switchMerchantsTab`函数，使用更具体的选择器，只操作对应层级的Tab元素。

#### 技术实现

**修改switchProfitTab函数：**
```javascript
function switchProfitTab(tabName) {
    currentProfitTab = tabName;
    
    // 只更新分成管理Tab内部的子Tab
    const profitTabContent = document.getElementById('merchants-profit-tab-content');
    if (!profitTabContent) return;
    
    profitTabContent.querySelectorAll('.profit-tab').forEach(tab => {
        tab.classList.remove('active');
    });
    // ... 只操作profitTabContent内部的元素
}
```

**修改switchMerchantsTab函数：**
```javascript
function switchMerchantsTab(tabName) {
    currentMerchantsTab = tabName;
    
    // 只更新主Tab（使用更具体的选择器）
    const mainCard = document.querySelector('#page-content > .card');
    const mainTabs = mainCard.querySelector('.profit-tabs');
    // ... 只操作mainCard内部的元素
}
```

## [1.7.6] - 2026-01-15

### 🎨 性能优化：分成管理页面防闪烁优化

#### 问题描述
在【分成管理】页面的【品牌总部分成设置】Tab中，页面内容会每5秒闪现一次，影响用户体验。

#### 问题原因
- 系统有一个定时器每5秒自动刷新当前页面
- 目的是确保数据实时同步
- 但分成管理页面的数据（profitRules和distributors）通常不会频繁变化
- 频繁刷新导致页面内容闪烁

#### 解决方案
优化定时刷新逻辑，当用户在【分成管理】页面时，跳过自动刷新，避免不必要的页面重新渲染。

#### 技术实现

**修改listenStorageChanges函数：**
```javascript
function listenStorageChanges() {
    window.addEventListener('storage', (e) => {
        if (['applications', 'orders', 'deposits', 'products', 'goods'].includes(e.key)) {
            renderPage(currentPage);
            showToast('数据已更新', 'success');
        }
    });
    // 定期刷新数据，确保实时同步
    // 但分成管理页面不需要频繁刷新，避免闪烁
    setInterval(() => {
        if (currentPage !== 'merchants-profit') {
            renderPage(currentPage);
        }
    }, 5000);
}
```

#### 优化效果

**性能提升**
- ✅ 分成管理页面不再每5秒刷新
- ✅ 消除了页面内容闪烁问题

#### 修复iframe白屏问题

- 兼容iframe和直接访问两种方式
