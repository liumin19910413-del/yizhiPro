# MG端审核流程进一步简化说明

## 更新日期
2026年1月22日（第二次更新）

## 更新概述
在MG管理后台的"商户入驻-入驻审批"模块中，进一步简化了审核申请流程：
1. 去掉"品牌总部"信息显示
2. 去掉"AI检测结果"
3. 新增"营业执照判断是否符合所需经营范围"单选框
4. 选择"不符合"时点击审核通过会报错

## 修改位置

**路径**: MG管理后台 → 商户入驻 → 入驻审批 → 点击"审核"按钮 → 审核申请弹窗

## 详细修改内容

### 1. 去掉品牌总部信息显示

**删除内容：**
```javascript
${app.brandMerchantId ? `
<div class="form-group">
    <label>品牌总部：</label>
    <span>${app.brandMerchantId} (分成${app.brandRatio}%)</span>
</div>
${receiverInfo}
${receiverApplicationInfo}
` : ''}
```

**原因：**
- 简化审核界面
- 品牌总部信息对审核决策无直接影响
- 与Lite端简化流程保持一致

### 2. 去掉AI检测结果

**删除内容：**
```javascript
<div class="form-group">
    <label>AI检测结果：</label>
    <div style="padding: 10px; background: ${autoRejectReason ? '#fff2e8' : '#f6ffed'}; border: 1px solid ${autoRejectReason ? '#ffbb96' : '#b7eb8f'}; border-radius: 4px;">
        ${autoRejectReason ? `<div style="color: #ff4d4f;">❌ ${autoRejectReason}</div>` : '<div style="color: #52c41a;">✓ 经营范围符合要求，已开通分账功能</div>'}
    </div>
</div>
```

**原因：**
- AI检测结果仅供参考，最终需要人工判断
- 简化界面，减少信息干扰
- 改为人工判断更准确

### 3. 新增营业执照判断单选框

**新增内容：**
```javascript
<div class="form-group">
    <label>营业执照判断是否符合所需经营范围：</label>
    <div style="display: flex; gap: 20px; margin-top: 10px;">
        <label style="display: flex; align-items: center; gap: 5px; cursor: pointer;">
            <input type="radio" name="scopeValid" value="true" checked>
            <span>符合</span>
        </label>
        <label style="display: flex; align-items: center; gap: 5px; cursor: pointer;">
            <input type="radio" name="scopeValid" value="false">
            <span>不符合</span>
        </label>
    </div>
</div>
```

**特点：**
- 单选框，只能选择"符合"或"不符合"
- 默认选中"符合"
- 必须由审核人员手动判断

### 4. 新增验证逻辑

**在 `approveApp` 函数中添加：**
```javascript
function approveApp(appId) {
    const scopeValid = document.querySelector('input[name="scopeValid"]:checked').value === 'true';
    
    // 验证营业执照经营范围
    if (!scopeValid) {
        showToast('营业执照经营范围不符合所需经营范围，只能拒绝让商户进行工商变更', 'error');
        return;
    }
    
    // ... 其他验证和处理逻辑
}
```

**验证规则：**
- 如果选择"不符合"，点击"审核通过"会报错
- 错误提示："营业执照经营范围不符合所需经营范围，只能拒绝让商户进行工商变更"
- 必须选择"拒绝"按钮来拒绝申请

## 业务流程

### 场景1：营业执照符合要求

1. 审核人员查看营业执照
2. 判断经营范围符合要求
3. 选择"符合"（默认已选中）
4. 填写分成比例区间
5. 填写审核备注
6. 点击"审核通过"
7. ✅ 审核成功，申请状态变为"待签约"

### 场景2：营业执照不符合要求

1. 审核人员查看营业执照
2. 判断经营范围不符合要求
3. 选择"不符合"
4. 尝试点击"审核通过"
5. ❌ 系统报错："营业执照经营范围不符合所需经营范围，只能拒绝让商户进行工商变更"
6. 审核人员点击"拒绝"按钮
7. 填写拒绝原因（建议商户进行工商变更）
8. ✅ 拒绝成功，申请状态变为"已拒绝"

## 代码修改总结

### 修改的函数

1. **showAuditModal(appId)** - 审核弹窗
   - 删除品牌总部信息显示
   - 删除AI检测结果
   - 删除自动拒绝逻辑
   - 新增营业执照判断单选框
   - 简化弹窗按钮（统一显示审核通过/拒绝/取消）

2. **approveApp(appId)** - 审核通过
   - 新增营业执照判断验证
   - 选择"不符合"时阻止审核通过

3. **删除的函数**
   - `autoRejectApp(appId, reason)` - 自动拒绝功能已移除

```javascript
const hasSubAccount = merchant.hasSubAccount;

```

## 用户体验改进

### 审核人员视角

**改进前：**
- 看到AI检测结果，可能过度依赖
- 品牌总部信息干扰审核决策
- 自动拒绝功能可能误判

**改进后：**
- 必须人工判断营业执照
- 界面更简洁，信息更聚焦
- 审核决策更准确

### 商户视角

**改进前：**
- 可能被系统自动拒绝
- 不清楚拒绝原因是AI还是人工

**改进后：**
- 所有审核都是人工审核
- 拒绝原因更明确
- 更有针对性的改进建议

## 注意事项

1. **人工判断责任**
   - 审核人员需要仔细查看营业执照
   - 准确判断经营范围是否符合要求
   - 不能依赖AI自动判断

2. **拒绝原因说明**
   - 选择"不符合"时，建议在拒绝原因中说明：
   - "营业执照经营范围不符合要求，需包含'化妆品零售'或'化妆品销售'，请进行工商变更后重新申请"

3. **浏览器缓存**
   - 测试时记得强制刷新（Cmd + Shift + R）
   - 或在开发者工具中勾选"Disable cache"
