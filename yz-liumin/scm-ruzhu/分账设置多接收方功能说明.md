# 分账设置多接收方功能说明

## 📅 更新时间
2025-01-20

## 🎯 功能概述

重新设计分账设置功能，支持为一个商户配置多个分账接收方，每个接收方可以设置不同的分成比例。使用表格形式展示和编辑，提供更灵活的分账配置能力。

## 🔄 主要变更

### 1. 从单接收方改为多接收方

**更新前：**
- 一个商户只能配置一个分账接收方
- 一个固定的分成比例
- 使用下拉框和输入框

**更新后：**
- ✅ 一个商户可以配置多个分账接收方
- ✅ 每个接收方可以设置独立的分成比例
- ✅ 使用表格形式展示和编辑
- ✅ 实时计算总分成比例
- ✅ 支持添加、修改、删除接收方

### 2. 界面重新设计

## 📊 数据结构变更

### 申请记录（applications）

**更新前：**
```javascript
{
  profitReceiver: 'PR001',  // 单个接收方ID
  brandRatio: 10            // 单个分成比例
}
```

**更新后：**
```javascript
{
  profitSettings: [         // 多个接收方配置
    { receiverId: 'PR001', ratio: 10 },
    { receiverId: 'PR002', ratio: 5.5 }
  ],
  brandRatio: 15.5          // 总分成比例（自动计算）

```

## 🎨 功能详情

### 1. 查看分账设置

**显示内容：**
- 商户基本信息
- 分账接收方列表（表格形式）
- 每个接收方的详细信息和分成比例
- 总分成比例统计

**权限：**
- 有 `view` 权限即可查看
- 只读模式，不能修改

### 2. 添加分账接收方

**操作流程：**
1. 点击"+ 添加接收方"按钮
2. 弹出选择接收方对话框
3. 从可用列表中选择接收方
4. 输入分成比例（0-100%）
5. 点击"确定"添加

**验证规则：**
- 必须选择接收方
- 分成比例必须在0-100之间
- 不能重复添加同一个接收方
- 总分成比例不能超过100%

### 3. 修改分成比例

**操作方式：**
- 直接在表格的输入框中修改
- 实时更新总分成比例
- 支持小数输入（如10.5%）

**验证规则：**
- 分成比例必须在0-100之间
- 总分成比例不能超过100%

### 4. 删除分账接收方

**操作流程：**
1. 点击接收方行的"删除"按钮
2. 确认删除
3. 从列表中移除

**限制：**
- 至少保留一个接收方
- 删除后自动重新计算总分成比例

### 5. 保存设置

**验证规则：**
- 至少有一个分账接收方
- 总分成比例不能超过100%
- 每个接收方的比例必须有效

**保存内容：**
- 所有接收方配置
- 自动计算并保存总分成比例
- 更新时间

## 🔐 权限控制

### 三种权限级别

| 权限 | 配置 | 功能 |
|------|------|------|
| **不可见** | 无 `view` 权限 | 不显示"分账设置"按钮 |
| **只读** | 有 `view` 权限 | 查看表格，不能修改 |
| **可编辑** | 有 `edit` 权限 | 添加、修改、删除接收方 |

### 权限配置示例

```javascript
// 店长 - 完整权限
{
  'supply-chain-profit-setting': ['view', 'edit']
}

// 财务 - 只读权限
{
  'supply-chain-profit-setting': ['view']
}

// 收银员 - 不可见
{
  // 没有 supply-chain-profit-setting 权限
}
```

## 🎯 显示条件

**更新前：**
- 必须有品牌总部（`brandMerchantId`）
- 必须有分成比例（`brandRatio > 0`）

**更新后：**
- ✅ 只要有品牌总部（`brandMerchantId`）即可
- ✅ 不再要求必须有分成比例
- ✅ 支持从0开始配置

## 💡 智能提示

### 1. 总分成比例提示

- **等于100%**：绿色显示，表示配置完整
- **小于100%**：黄色提示"总分成比例未达到100%"
- **大于100%**：红色警告"总分成比例超过100%，请调整"

### 2. 空状态提示

- 无接收方时：显示"暂无分账接收方设置，点击右上角'添加接收方'开始配置"
- 无可用接收方时：显示"品牌总部暂未配置分账接收方，请联系品牌总部管理员添加"

### 3. 权限提示

- 只读模式：显示"您当前只有查看权限，如需修改请联系管理员"

## 📝 技术实现

### 核心函数

1. **showProfitSettingModal()** - 显示分账设置弹窗
   - 支持多接收方表格展示
   - 区分可编辑和只读模式
   - 实时计算总分成比例

2. **addProfitReceiver()** - 添加接收方
   - 弹出选择对话框
   - 过滤已添加的接收方
   - 验证输入

3. **updateProfitRatio()** - 更新分成比例
   - 实时验证
   - 自动更新总计
   - 颜色提示

4. **removeProfitReceiver()** - 删除接收方
   - 确认对话框
   - 重新渲染表格
   - 更新总计

5. **saveProfitSettings()** - 保存设置
   - 完整验证
   - 保存到数据库
   - 更新总分成比例

### 全局变量

```javascript
window.currentProfitSettings  // 当前编辑的设置
window.currentAppId           // 当前申请ID
window.availableReceivers     // 可用接收方列表
```

## ⚠️ 注意事项

1. **数据兼容性**：保留 `brandRatio` 字段用于向后兼容
2. **总比例限制**：总分成比例不能超过100%
3. **至少一个接收方**：保存时必须至少有一个接收方
4. **重复添加**：不能重复添加同一个接收方
5. **实时验证**：输入时实时验证和提示
