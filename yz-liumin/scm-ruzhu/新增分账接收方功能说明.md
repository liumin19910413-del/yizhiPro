# 新增分账接收方功能说明

## 功能概述
新增分账接收方是一个独立的功能，用于创建分账接收方账户。创建成功后，该接收方会生成一个唯一ID，可以在后续的分账设置中被选择和关联到具体的商户号。

## 核心逻辑

### 1. 分账接收方的独立性
- **分账接收方是独立实体**：不依赖于商户号，可以单独创建
- **创建后可复用**：一个分账接收方可以被多个商户号选择使用
- **先创建后关联**：流程是"创建接收方 → 在分账设置中选择接收方 → 关联到商户号"

### 2. 操作流程

#### 场景1：初次申请入驻时
1. 用户进入"申请入驻-连锁"流程
2. 选择品牌总部商户号
3. 点击"+ 新增分账接收方"按钮
4. 填写完整的接收方信息（对公/对私账户）
5. 提交后，系统创建一个新的分账接收方记录
6. 返回分账设置页面，在下拉框中选择刚创建的接收方
7. 设置分成比例并关联到具体商户号

#### 场景2：编辑已拒绝申请时
1. 用户进入"入驻管理-申请记录"
2. 找到"已拒绝"状态的申请，点击"编辑资料"
3. 在分账设置区域，勾选需要设置分账的商户号
4. 点击"+ 新增分账接收方"按钮（无需先选择商户号）
5. 填写完整的接收方信息
6. 提交后，系统创建新的分账接收方
7. 在表格中为已勾选的商户号选择接收方并设置比例

## 核心函数

### showAddEditReceiverModal()
显示新增分账接收方弹窗，包含完整的对公/对私账户表单。

**特点**：
- 不需要任何参数
- 不依赖于商户号选择
- 创建的是独立的接收方实体

### confirmAddEditReceiver()
提交新增分账接收方表单。

**验证逻辑**：
1. 基本信息验证（名称、手机号）
2. 根据账户类型验证对应字段
3. 文件上传验证（大小不超过2M）
4. 创建接收方记录并保存到 DataStore

### switchReceiverTab(tab)
切换对公/对私账户Tab，显示/隐藏对应的表单字段。

### handleFileUpload(input, previewId)
处理文件上传，验证文件大小并显示文件名预览。

## 与分账设置的关系

1. **创建阶段**：新增分账接收方 → 生成接收方ID
2. **关联阶段**：在分账设置中选择接收方 → 关联到商户号 → 设置分成比例
3. **数据存储**：
   - 接收方数据存储在 `profitReceivers` 表
   - 分账设置存储在申请记录的 `profitSettings` 数组中

```javascript
// 分账设置数据结构
profitSettings: [
    {
        merchantId: 'M001',      // 商户号
        receiverId: 'PR123',     // 接收方ID（引用）
        ratio: 30                // 分成比例
    }
]
```

## 修复说明

### 问题
之前的实现要求用户先选择商户号才能新增分账接收方，这是不合理的。

### 解决方案
1. 移除了"选择商户号"下拉框
2. 移除了 `confirmAddEditReceiver()` 中对商户号的验证
3. 移除了 `showAddEditReceiverModal()` 中对已选商户号的检查
4. 简化了 `showAddReceiverModal()` 和 `showAddReceiverModalWithMerchantId()` 函数

### 正确的使用流程
1. 点击"+ 新增分账接收方"（无需任何前置条件）
2. 填写完整的接收方信息
3. 提交创建接收方
4. 在分账设置表格中选择刚创建的接收方
5. 为选中的商户号设置分成比例
