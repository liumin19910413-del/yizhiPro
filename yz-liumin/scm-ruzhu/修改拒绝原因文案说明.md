# 修改拒绝原因文案说明

## 📅 修改时间
2026-01-20

## 🎯 修改内容

在Lite商户端的"营销应用-店商供应链-入驻管理"中，修改"已拒绝"状态申请的拒绝原因显示文案。

## 📝 修改位置

### 1. 编辑资料弹窗
**位置**：点击"已拒绝"状态申请的【编辑资料】按钮

**修改前**：
```
❌ 上次申请被拒绝原因：
[显示数据库中的 app.auditRemark 或 app.rejectReason 或 '未提供具体原因']
```

**修改后**：
```
❌ 上次申请被拒绝原因：
您申请的商户号营业执照的经营范围还未更新，请您按申请入驻时提醒你要的经营范围去工商进行变更，变更成功后再进行提交。
```

### 2. 申请详情弹窗
**位置**：点击"已拒绝"状态申请的【查看原因】按钮

**修改前**：
```
❌ 拒绝原因：
[显示数据库中的 app.auditRemark 或 app.rejectReason 或 '未提供具体原因']
```

**修改后**：
```
❌ 拒绝原因：
您申请的商户号营业执照的经营范围还未更新，请您按申请入驻时提醒你要的经营范围去工商进行变更，变更成功后再进行提交。
```

## 💡 说明

### 1. 固定文案
- 拒绝原因现在显示为固定的文案，不再从数据库中读取
- 所有"已拒绝"状态的申请都会显示相同的拒绝原因

### 2. 文案内容
拒绝原因文案强调了：
- 问题：营业执照的经营范围还未更新
- 参考：按申请入驻时提醒的经营范围
- 操作：去工商进行变更
- 要求：变更成功后再进行提交

### 3. 适用场景
- 编辑已拒绝的申请时
- 查看已拒绝申请的详情时

## 💡 注意事项

1. **固定文案**：拒绝原因现在是硬编码的固定文案，不会根据实际的拒绝原因变化
2. **数据库字段**：虽然数据库中可能存储了 `app.auditRemark` 或 `app.rejectReason`，但界面上不再显示这些内容
3. **统一体验**：所有已拒绝的申请都会看到相同的拒绝原因提示

## 🔄 如需恢复

如果将来需要恢复显示数据库中的实际拒绝原因，只需将代码改回：

```javascript
// 恢复为显示数据库中的原因
<p style="color: #666;">${app.auditRemark || app.rejectReason || '未提供具体原因'}</p>
```
