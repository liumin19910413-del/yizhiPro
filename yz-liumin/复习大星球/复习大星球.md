## 1. 产品概述

### 1.1 产品名称

复习小星球

### 1.2 产品定位

复习小星球是一款基于孩子学校教材的课后复习巩固应用。

它不是补课工具，不是单纯刷题工具，也不是以闯关游戏为主的儿童娱乐产品。它的核心是把孩子学校已经学过的教材内容，拆成每天可执行、可验证、可复盘的复习任务。

产品要帮助孩子做到：

1. 会读：生字、词语、课文段落能读出来。
2. 会认：能认字、认拼音、认词语。
3. 会写：能把本课生字词写对。
4. 会默：听系统读，能在本子上默写出来。
5. 会复盘：错字、错词、错题自动进入再练。

### 1.3 一句话说明

每天跟着课本复习一点点，把学校学过的内容真正记牢。

### 1.4 核心理念

孩子端永远只看到“今天要做什么”。  
家长端负责设置学校进度、开启期末总复习、查看学习报告。  
系统负责基于教材出题、读题、收集录音/照片、自动生成报告和再练任务。

## 2. 目标用户与场景

### 2.1 孩子

MVP 面向小学低年级孩子，首期以二年级语文复习为主。

孩子特点：

1. 不适合复杂菜单。
2. 能接受短步骤任务。
3. 需要即时反馈和奖励。
4. 容易因为答错受挫，因此不做扣分惩罚。
5. 读、写、默写都需要明确指令和简单操作。

孩子端诉求：

1. 一眼知道今天复习哪一课。
2. 点开始后，一步一步完成。
3. 不需要自己规划。
4. 做对有小红花。
5. 做错知道正确答案，并后面再练。

### 2.2 家长

家长是复习计划和学习结果的负责人。

家长诉求：

1. 能设置孩子学校当前学到哪一课。
2. 平时自动生成课后复习。
3. 期末能生成整册复习计划。
4. 不希望每天人工批改每一道题。
5. 需要看到每日学习报告。
6. 需要知道孩子哪里不会、明天会加练什么。
7. 奖励要简单，能和家庭奖励挂钩。

## 3. 产品模式

复习小星球支持两种模式。

### 3.1 平时复习模式

平时复习是默认主模式。

家长设置孩子学校当前学到哪一课，系统根据复习规则生成今日任务。

推荐规则：

```text
学校当前学到：第3课《开满鲜花的小路》
系统今日复习：第2课或第3课
默认建议：慢1课复习
```

平时复习的目标是每天轻量巩固，不抢学校作业时间。

### 3.2 期末总复习模式

期末模式用于整本教材都学完后，按考试日期和复习范围生成复习计划。

家长可设置：

1. 复习教材。
2. 复习范围。
3. 考试日期。
4. 每天复习几课。
5. 是否开启错字错词集中复习。
6. 是否开启朗读抽查。
7. 是否开启全册小测。

孩子端仍然只显示“今日复习”，不展示复杂日历。

## 4. 每课任务结构

每课固定 5 类任务。

### 4.1 读一读

目标：确认孩子能读出生字和词语。

内容：

1. 本课会认字。
2. 本课会写字。
3. 本课重点词语。

交互：

1. 屏幕显示需要读的生字和词语。
2. 孩子点击开始录音。
3. 读完后停止录音。
4. 可试听、重录、提交。
5. 未录音不能完成。

结果：

1. 录音提交成功后获得小红花。
2. 系统可进行朗读识别。
3. 识别不清的字词进入学习报告。

### 4.2 认一认

目标：像语文老师一样检查字音、字形、词语和课文理解。

题型：

1. 汉字选拼音。
2. 拼音选汉字。
3. 组词选择。
4. 词语认读。
5. 词义选择。
6. 词语搭配。
7. 课文简单理解。

规则：

1. 客观题自动判对错。
2. 答对得小红花。
3. 答错不扣分。
4. 答错后必须弹出正确答案。
5. 点击“我知道了”后进入下一题。
6. 错题进入再练本。

禁止规则：

1. 不允许出现“汉字下面选同一个汉字”的无效题。
2. 没有拼音数据时，不硬凑拼音题。
3. 低年级题干必须短，一题只考一个点。

### 4.3 写一写

目标：练字形、书写和词语书写。

内容：

1. 本课会写字。
2. 本课重点词语。

交互：

1. 屏幕可以显示生字和词语。
2. 孩子照着写到本子上。
3. 写完后拍照提交。
4. 可查看照片、重新拍照、提交。

结果：

1. 拍照提交成功获得小红花。
2. 系统识别照片。
3. 识别结果进入学习报告。
4. 错字、书写不清的字词进入再练本。

### 4.4 默一默

目标：防作弊地完成听写/默写。

内容：

1. 本课会写字。
2. 本课重点词语。

孩子端规则：

1. 屏幕不能显示要默写的字词答案。
2. 屏幕只显示“默写一、默写二、默写三”。
3. 孩子点击播放，系统读出隐藏字词。
4. 孩子听后写到本子上。
5. 全部写完后拍照提交。

后台隐藏清单示例：

```text
默写一：鲜花
默写二：原来
默写三：大叔
默写四：正巧
默写五：惊奇
默写六：礼物
```

家长端规则：

1. 家长不需要逐题人工审核。
2. 系统识别照片并对比隐藏清单。
3. 学习报告展示默写结果。
4. 低置信度结果标记“建议家长抽查”。

结果：

1. 拍照提交成功获得小红花。
2. 默写全对可获得额外小红花。
3. 错词进入再练本。

### 4.5 读课文

目标：确认孩子完成课文片段朗读。

内容：

1. 本课某一个自然段。
2. 不要求整篇课文。
3. 可优先选课文重点段落。

交互：

1. 屏幕展示要读的段落。
2. 孩子点击开始录音。
3. 读完后停止录音。
4. 可试听、重录、提交。
5. 未录音不能完成。

结果：

1. 录音提交成功获得小红花。
2. 系统识别朗读清晰度。
3. 识别不清或漏读进入报告。

## 5. 出题规则

产品必须具备“语文老师式出题”能力，而不是简单把教材内容搬到页面上。

### 5.1 出题能力维度

每课题目要覆盖：

1. 字音：看汉字选拼音、看拼音选汉字、易错音辨析。
2. 字形：形近字、错别字、组词判断。
3. 词语：词语认读、词义选择、近义词、反义词、搭配。
4. 句子：简单句意理解。
5. 课文：围绕课文内容做基础理解题。

### 5.2 每课推荐题量

平时复习每课推荐：

```text
字音题：3题
拼音选字：2题
组词题：2题
词义/搭配题：2题
课文理解题：1题
```

期末复习可按复习强度增加题量。

### 5.3 示例题

```text
“裹”的正确读音是？
[guǒ] [guō] [kě]
```

```text
这个拼音对应哪个字？
guǒ
[裹] [粒] [破]
```

```text
“正巧”的意思更接近哪个？
[刚好] [很难过] [跑得快]
```

```text
《开满鲜花的小路》里，美好的礼物最后是什么？
[花籽] [邮包] [小路上的鲜花]
```

### 5.4 错题处理

答错后展示：

```text
答错啦
正确答案是：guǒ
这题已经放进再练本

[我知道了]
```

孩子必须点击“我知道了”后才进入下一题。

## 6. 奖励体系

奖励体系只保留一种奖励货币：小红花。

### 6.1 发放规则

1. 客观题答对：+1 朵。
2. 答错：+0，不扣分，进入再练。
3. 读一读录音提交：+1 朵。
4. 读课文录音提交：+1 朵。
5. 朗读识别/报告通过：额外 +1 朵。
6. 写一写拍照提交：+1 朵。
7. 写字识别/报告通过：额外 +1 朵。
8. 默一默拍照提交：+1 朵。
9. 默写全对：额外 +3 朵。
10. 默写大部分对：额外 +1 朵。
11. 完成今日全部复习：额外 +2 朵。
12. 连续 3 天完成：额外 +3 朵。
13. 连续 7 天完成：额外 +7 朵。

### 6.2 兑换规则

1. 小红花可兑换家长设置的奖品。
2. 孩子小红花够了可以申请兑换。
3. 家长确认后扣减小红花。
4. 系统保留兑换记录。

### 6.3 不做的奖励规则

1. 不做多币种。
2. 不做排行榜。
3. 不做答错扣分。
4. MVP 不做复杂宠物养成。
5. 不让奖励商城抢走复习主线。

## 11. 自动识别与报告规则

### 11.1 朗读识别

MVP 可以先记录录音提交，后续接入语音识别。

识别维度：

1. 是否有录音。
2. 录音时长是否合理。
3. 是否读出目标字词。
4. 是否漏读。
5. 是否明显读错。

报告展示：

```text
字词朗读：已提交
识别结果：基本清楚
建议再练：裹、漏
```

### 11.2 写字识别

MVP 可先完成照片提交与报告占位，后续接入 OCR/AI 识别。

识别维度：

1. 是否提交照片。
2. 图片是否清晰。
3. 目标字词是否完整。
4. 是否存在错别字。
5. 是否需要重写。

### 11.3 默写识别

默写识别必须和隐藏默写清单对比。

结果分类：

1. 全对。
2. 大部分正确。
3. 错得多。
4. 识别不清，建议家长抽查。

报告展示：

```text
默写 6 个
写对 5 个
错词：正巧
识别为：正桥
```

## 13. 验收标准

### 13.1 平时复习

1. 家长可设置学校当前学到哪一课。
2. 系统可按当天课/慢1课/慢2课生成今日复习。
3. 孩子首页只显示今日复习内容。
4. 孩子可按 5 个任务顺序完成复习。

### 13.2 期末复习

1. 家长可选择复习范围和考试日期。
2. 系统可生成复习日历。
3. 家长可启用期末模式。
4. 孩子端仍只显示今日复习任务。

### 13.3 默写防作弊

1. 孩子端默写页不得展示答案。
2. 只能显示“默写一、默写二、默写三”。
3. 点击播放后系统读出隐藏内容。
4. 家长报告能看到隐藏清单与识别结果。

### 13.4 录音任务

1. 读一读必须录音提交。
2. 读课文必须录音提交。
3. 未录音不能完成任务。
4. 录音可试听、重录、提交。

### 13.5 拍照任务

1. 写一写必须拍照提交。
2. 默一默必须拍照提交。
3. 可重新拍照。
4. 提交后进入报告识别流程。

### 13.6 出题质量

1. 不出现汉字选同一个汉字的无效题。
2. 有拼音数据时出汉字选拼音、拼音选汉字。
3. 无拼音数据时不硬凑拼音题。
4. 每题只考一个语文能力点。
5. 答错必须展示正确答案。

### 13.7 奖励

1. 小红花余额展示正确。
2. 答对题目奖励 +1。
3. 提交录音/照片按规则奖励。
4. 今日完成额外奖励。
5. 连续完成奖励。
6. 可兑换家长设置的奖品。
7. 兑换后扣减小红花并生成记录。

## 14. 关键结论

复习小星球的核心闭环是：

```text
教材内容结构化
↓
家长设置学校进度或期末计划
↓
系统生成今日复习
↓
孩子完成读、认、写、默、读课文
↓
系统收集录音/照片/答题结果
↓
自动生成学习报告
↓
错字错词进入再练本
↓
小红花激励孩子坚持
```

产品主结构必须围绕复习效果，而不是围绕闯关游戏。

孩子端要简单。  
家长端要省心。  
系统要像语文老师一样会出题、会读题、会记录、会复盘。
