# 渠道奖的业务逻辑说明

## 1. 功能定位

渠道奖是面向分销合伙人体系的奖励能力，用于基于合伙人名下两条核心渠道的业绩表现，计算其可获得的渠道佣金。

该能力解决的问题是：

- 平台需要按月归属渠道奖数据
- 平台需要按 `T+1` 的方式，展示截至昨日的最新渠道业绩和佣金情况
- 平台需要识别每个合伙人的第一大市场和第二大市场
- 平台需要基于两大市场的业绩对比结果，计算该合伙人的渠道奖佣金

本方案明确约定：

`渠道奖按合伙人直推下级所形成的渠道业绩进行统计，按月归属，按 T+1 展示截至昨日的最新累计结果，分佣基数只取第一大市场和第二大市场。`

## 2. 功能需求

### 2.1 菜单挂载位置

结合现有分销系统菜单结构，新增两个同级菜单：

- 在 `奖励设置` 下，`育才奖设置` 下面增加 `渠道奖设置`
- 在 `分红结算` 下面增加 `渠道奖统计`

要求：

- `渠道奖设置` 与 `育才奖设置` 为同级菜单
- `渠道奖统计` 与 `分红结算` 为同级菜单入口逻辑保持一致，页面风格沿用现有分销系统后台样式

### 2.2 组织适用范围

- 该功能 `连锁总部` 和 `分店` 都要具备
- `连锁总部` 可统一配置渠道奖规则，并支持将配置分发到分店
- `连锁总部` 可查看全部门店的渠道奖统计结果
- `分店` 可查看本门店范围内的渠道奖设置与渠道奖统计结果
- 因总部存在跨门店查看诉求，`渠道奖统计` 页面筛选区需比单店版本增加 `门店筛选`

### 2.3 统计对象

- 统计对象为分销体系中的合伙人
- 每个合伙人以自己的直推下级作为市场划分起点
- 每个直推下级及其下方全部层级成员共同构成一条市场渠道

### 2.4 核心概念

- `直推下级`：合伙人直接邀请或推荐的下一级合伙人
- `渠道`：某个直推下级本人加其下方所有层级成员形成的完整网络
- `渠道业绩`：该渠道内所有成员产生的有效订单业绩之和
- `第一大市场`：合伙人所有直推下级中，渠道业绩最高的那条渠道
- `第二大市场`：合伙人所有直推下级中，渠道业绩第二高的那条渠道
- `渠道长`：某条渠道对应的直推下级本人
- `渠道奖`：基于第一大市场和第二大市场业绩计算出的渠道佣金

### 2.5 使用要求

- 系统需支持在 `渠道奖设置` 页面统一配置渠道奖佣金比例
- 系统需支持总部将渠道奖配置分发到分店
- 系统需通过 `开启/关闭渠道奖机制` 控制该功能是否生效
- 仅在开启后，系统才执行每日渠道奖统计计算
- 未开启时，系统不执行每日统计计算
- 系统需支持在设置页清晰说明该奖项的业务价值、触发条件和计算规则
- 系统需支持在 `渠道奖统计` 页面查看按月归属的渠道奖统计结果
- 系统需支持在统计页展示截至昨日的最新统计截止时间
- 系统需支持总部按门店筛选统计结果
- 系统需支持按月份筛选、按合伙人姓名和手机号搜索
- 系统需支持导出渠道奖统计结果
- 系统需支持默认按佣金金额从高到低排序展示

### 2.6 结果要求

- 后台可查看全部合伙人的渠道奖统计结果
- 后台可查看每个合伙人的第一大市场、第二大市场及对应佣金计算结果
- 总部可查看每个门店的渠道奖统计结果
- 后台可导出按月归属的渠道奖结果报表

## 3. 业务逻辑

### 3.1 渠道业绩计算逻辑

某个直推下级的渠道业绩，等于该直推下级本人业绩，加上其下方所有层级成员的业绩总和。

即：

`渠道业绩 = 渠道长本人业绩 + 该渠道下所有成员业绩`

渠道范围按无限层级向下递归统计，不只统计一层或两层。

### 3.2 第一大市场与第二大市场判定逻辑

- 对某个合伙人，先统计其所有直推下级对应的渠道业绩
- 按渠道业绩从高到低排序
- 排名第 1 的为 `第一大市场`
- 排名第 2 的为 `第二大市场`
- 若直推下级超过 2 个，仅取前 2 名参与渠道奖计算

需要注意：

- 排名第 3 及之后的渠道，不参与渠道奖佣金基数计算
- 但这些渠道仍属于该合伙人的渠道网络组成部分

### 3.3 渠道奖触发条件

渠道奖必须同时满足以下条件才可计算：

- 合伙人名下至少有 2 个直推下级
- 且这 2 个直推下级对应的渠道业绩都大于 0

若不满足以上条件，则：

`当前统计结果中的渠道奖佣金 = 0`

### 3.4 渠道奖计算规则

设：

- `大市场业绩 = 第一大市场业绩与第二大市场业绩中的较大值`
- `小市场业绩 = 第一大市场业绩与第二大市场业绩中的较小值`
- `佣金比例 = 后台统一配置的比例`

计算逻辑如下：

```text
如果 大市场业绩 ÷ 小市场业绩 > 2
则：
佣金基数 = 小市场业绩
渠道奖佣金 = 小市场业绩 × 佣金比例

如果 大市场业绩 ÷ 小市场业绩 ≤ 2
则：
佣金基数 = 大市场业绩 ÷ 2
渠道奖佣金 = (大市场业绩 ÷ 2) × 佣金比例
```

### 3.5 规则理解

该规则的核心是：

`按两条市场中相对均衡的那部分业绩作为佣金基数。`

具体含义为：

- 当两条市场差距过大时，只按小市场业绩计算，避免单边市场过强时奖励失真
- 当两条市场相对均衡时，按大市场一半作为基数，体现双市场平衡发展

### 3.6 佣金比例规则

- 渠道奖佣金比例采用统一配置
- 同一套渠道奖规则下，所有合伙人按统一佣金比例参与计算
- 系统在计算渠道奖时，直接读取当前生效的统一佣金比例

### 3.7 统计口径

- 渠道奖按 `月` 作为归属维度
- 页面筛选维度为 `月份`
- 渠道奖机制开启后，渠道业绩与佣金结果按 `T+1` 口径更新
- 即今天早上可看到 `截止昨天` 的第一大市场、第二大市场业绩及当前佣金情况
- 渠道奖机制未开启时，不触发每日统计计算，也不产生每日累计结果
- 页面必须明确展示 `最新统计截止时间`
- 业绩口径以有效订单实收金额为准
- 若订单发生退款，退款金额在退款发生时冲减对应统计结果

### 3.8 排序逻辑

- `渠道奖统计` 列表默认按 `佣金金额` 从高到低排序
- 当佣金金额相同时，可再按合伙人姓名或生成时间作为次级排序

## 4. 业务流程

### 4.1 主流程

```text
商户在奖励设置中完成渠道奖规则配置
→ 开启渠道奖机制
→ 系统按当前统一佣金比例应用规则
→ 系统每日按 T+1 更新渠道奖统计数据
→ 今日早上生成截至昨日的最新市场业绩与佣金结果
→ 运营进入渠道奖统计页面按月份查看结果
→ 运营可按姓名、手机号查询目标合伙人
→ 系统按佣金金额从高到低展示统计列表
→ 运营可导出当月渠道奖统计结果
```

### 4.2 计算流程

```text
读取某合伙人全部直推下级
→ 统计每个直推下级对应的渠道业绩
→ 按业绩排序
→ 取前 2 名作为第一大市场和第二大市场
→ 判断是否满足渠道奖计算条件
→ 判断大市场与小市场比值
→ 按命中规则计算佣金基数
→ 读取该合伙人等级对应佣金比例
→ 计算渠道奖佣金金额
→ 写入当前月份截至昨日的最新统计结果
```

### 4.3 异常流程

#### 4.3.1 直推市场不足

```text
系统统计某合伙人直推市场
→ 发现有效市场不足 2 条
→ 不触发渠道奖计算
→ 该合伙人当前佣金金额为 0
```

#### 4.3.2 第二大市场业绩为 0

```text
系统识别第一大市场和第二大市场
→ 第二大市场业绩为 0
→ 不触发正常比值计算
→ 该合伙人当前佣金金额为 0
```

#### 4.3.3 佣金比例未配置

```text
系统完成市场排名与佣金基数判断
→ 发现当前统一佣金比例未配置
→ 阻止生成正常佣金结果
→ 页面提示规则待配置
```

- 内容区使用白底卡片和表格
- 按钮、输入框、下拉框、表格风格与现有 `育才奖设置`、`分红结算` 页面保持一致

- 当开关开启时：
  - 支持维护统一佣金比例
  - 佣金比例输入项只允许输入合法百分比
  - 系统开始启用每日渠道奖统计计算
- 页面中的说明内容为辅助信息，需使用更小、更浅的样式，不抢配置主内容
- 页面不保留单独的大标题说明块
- 页面底部按钮风格与当前 `育才奖设置` 页面一致，按钮为 `复制给分店` 和 `保存设置`
