```
  ④ 选券弹窗「确认选择」（任何券变更）
  ⑤ 拨动积分开关
  ⑥ 切换支付方式

重算期间：
  → 金额数字区域显示 Loading 占位
  → 底部「提交订单」按钮置灰禁用
  → 寻优提示条显示"重新计算中… 🔄"

重算完成后：
  → 若结果优于用户手动选择：
    显示黄色提示条"⚠️ 当前方案比推荐多花 ¥XX  [恢复推荐方案]"
  → 若结果与推荐一致：
    提示条恢复蓝色"💡 已为您找到最省钱方案"
```

## 五、异常场景处理

| 异常场景 | 页面处理 |
|---------|---------|
| 计价接口超时 | Toast"计算中，请稍候"，按钮禁用，3s 后自动重试 |
| 用户方案比推荐贵 | 黄色提示条"⚠️ 当前比推荐方案多花¥XX [恢复推荐]" |
| 卡金不足支付全款 | Step7 标注"余额¥200，差额¥62 转现金" |
| 提交时券已失效 | 弹窗"满100减40券已失效，请重新选券" → 跳回选券 |
| 提交时赠送金余额变动 | 弹窗"赠送金余额已变更，已为您重新计算" → 自动重算 |
| 提交时商品已下架 | 弹窗"以下商品已下架：[商品名]，请返回购物车处理" |
| 提交时积分不足（并发） | 弹窗"积分已不足，已为您关闭积分抵扣并重新计算" |

> 📌 **设计原则备忘**
> - 所有金额展示：前端禁止本地估算，**必须来自后端计价接口返回**
> - 所有开关变更、券切换：统一通过 **防抖300ms** 后调用计价接口
> - 退款/分账：只读下单时的**品级流水快照**，绝不重算
> - 单卡绑定：折扣、赠送金、卡金**强制同源**，切卡后三者联动刷新
