本地商家合作收益分账需求PRD

⸻

一、需求背景

1.1 现状说明（线上已上线版本）

当前线上系统中，本地商家类订单的分佣处理存在以下问题：

商城推广佣金池（即平台毛利扣除技术服务费后的部分）中，直推（共建）30%、间推（店家）30%已按分佣规则处理，但其中40%的平台营销费部分，目前并未作为独立分佣账户处理：

- 若用户使用微信/支付宝等现金支付，这40%直接进入收款方账户，未单独拆分；
- 若用户使用积分支付，收款方侧没有对应的分账记录，这部分钱的归属不清晰。

原来的处理逻辑是将这40%视为商家收入的一部分整体处理，未做独立记账。

⸻

1.2 改造目标（本次需求为目标态）

随着业务发展，商家收益需要拆分得更加清晰，每一笔收入都需要有明确的名目和归属。因此本次需求的目标是：

1. 将商城推广佣金池中40%平台营销费从收款方收入中拆分出来，作为独立的系统虚拟账户（平台营销费账户）单独记账；
2. 本地商家的分成收入以"合作收益"名目独立记账，不与其他收入混合；
3. 实现每笔订单资金全链路可追溯、可对账、名目清晰。

本文档描述的是改造后的目标态，即未来希望达到的状态。

⸻

1.3 业务背景

平台商城对接本地合作商家（如水果店、旅行社、餐饮店、生活服务类商家等），用户通过商城下单消费后，订单金额需要按合作协议比例在本地商家与平台之间进行分成。

平台分成部分（即平台毛利）作为商城推广佣金池，按分佣规则分配给直推（共建）、间推（店家）和平台营销费账户。

本地商家收到的分成以"合作收益"名目进入其佣金钱包，走分销佣金体系统一结算。

当前系统已有分销佣金能力和统一钱包账本体系，本地商家业务需要复用该能力，核心新增点在于：

1. 支持按商家维度配置不同的合作分成比例；
2. 支持本地商家的"合作收益"收入名目进入统一钱包账本；
3. 平台毛利部分复用商城推广分佣规则（直推30% + 间推30% + 平台营销费40%）；
4. 支持多种支付方式（微信、支付宝、卡金、积分、混合支付）下收益计算不变。

⸻

二、需求目标

2.1 业务目标

1. 支持本地商家入驻平台商城后，按合作协议自动分账；
2. 支持本地商家的合作收益通过分销佣金体系统一结算；
3. 支持平台毛利部分复用商城推广分佣规则进行二次分配；
4. 支持不同商家配置不同分成比例；
5. 支持多种支付方式下的收益计算规则不变；
6. 支持退款、售后、冲正场景。

2.2 产品目标

1. 新增本地商家合作分成规则配置；
2. 新增"合作收益"收入名目；
3. 本地商家合作收益进入统一钱包账本；
4. 平台毛利走商城推广分佣规则（复用现有配置）；
5. 支持拉卡拉通道规则与业务账本解耦；
6. 支持技术服务费1.5%统一计提。

⸻

三、核心设计原则

3.1 收益计算按订单金额计算

本地商家所有收益和成本规则均以订单金额作为计算基数。

各方应分金额 = 订单金额 × 对应比例

不因用户使用微信、支付宝、卡金、积分或混合支付而改变各方应得金额。

⸻

3.2 支付方式影响资金来源，不影响应分规则

支付方式	是否影响收益计算	是否影响资金来源
微信 / 支付宝	否	是，现金收款资金
卡金	否	是，卡金沉淀资金或余额待分账资金池
积分	否	是，积分成本池或余额待分账资金池
混合支付	否	是，按支付构成匹配资金来源

⸻

3.3 拉卡拉规则不进入统一钱包账本

与399卡、980卡一致：拉卡拉手续费和商户主体预留金不进入统一钱包账本。

⸻

四、名词定义

名词	定义
订单金额	用户在商城购买本地商家商品/服务的实付金额
本地商家	与平台签订合作协议的本地合作商户
合作分成比例	本地商家从订单金额中获得的比例（按商家协议配置）
合作收益	本地商家从订单中获得的分成收入，收入名目为"合作收益"
平台毛利	订单金额扣除本地商家分成后的平台侧收入
平台毛利比例	订单金额中平台获得的比例 = 1 - 合作分成比例
商城推广佣金池	平台毛利作为佣金池，按分佣规则分配给各角色
技术服务费	按订单金额1.5%收取，不受支付方式影响

⸻

五、本地商家资金拆解规则

5.1 整体资金结构

订单金额 = 本地商家合作收益 + 平台毛利

平台毛利 = 技术服务费 + 商城推广佣金池

商城推广佣金池 = 平台毛利 - 技术服务费

⸻

5.2 资金拆解公式

项目	计算方式
本地商家合作收益	订单金额 × 合作分成比例
平台毛利	订单金额 × 平台毛利比例
技术服务费	订单金额 × 1.5%
商城推广佣金池	平台毛利 - 技术服务费

⸻

5.3 商城推广佣金池分配

商城推广佣金池复用现有分佣规则，通过分佣规则配置模块设置：

分佣角色	比例	账户类型
直推（共建）	30%	真实收益账户
间推（店家）	30%	真实收益账户
平台营销费	40%	系统虚拟账户

缺失角色兜底规则：

销售场景	直推（共建）	间推（店家）	平台营销费账户
共建销售	30%	30%	40%
店家直接销售	—（不存在）	30%	70%
无销售人	—	—	100%

⸻

本地商家合作收益	100 × 70%	70元

3. 规则修改后只影响新订单，不影响历史订单。
4. 支付成功时保存规则快照。
5. 技术服务费按订单金额全额1.5%计收，不受合作分成比例影响。

八、技术服务费规则

与399卡、980卡一致：

技术服务费总额 = 订单金额 × 1.5%

技术服务费计费基数为订单全额，不受支付方式影响。

拉卡拉手续费抵扣规则：

技术服务费补差 = 技术服务费总额 - 拉卡拉已扣手续费

若未使用拉卡拉支付：

技术服务费补差 = 技术服务费总额

九、订单收益生成流程

用户购买本地商家商品/服务
→ 订单支付成功
→ 读取该商家的合作分成规则
→ 按订单金额计算本地商家合作收益
→ 按订单金额计算技术服务费
→ 计算平台毛利
→ 计算商城推广佣金池 = 平台毛利 - 技术服务费
→ 判断销售人身份
→ 按分佣规则分配佣金池（直推/间推/平台营销费）
→ 生成各方收益明细
→ 本地商家合作收益进入商家钱包（名目：合作收益）
→ 共建/店家推广佣金进入对应钱包
→ 平台营销费进入系统虚拟账户
→ 技术服务费进入技术服务费账户

十一、售后退款与冲正规则

11.1 冲正原则

本地商家订单退款时，按退款比例冲正已生成的所有收益明细。

冲正金额 = 原明细金额 × 退款金额 / 原订单金额

全额退款时，所有明细全额冲正。

⸻

11.2 冲正处理

原明细状态	退款处理
待结算	直接生成负向冲正，减少待结算金额
可提现未提现	冻结并冲正可提现金额
提现中	标记风险，待提现结果后处理
已提现	生成负向收益明细，从后续收益中抵扣或进入追缴
系统虚拟账户明细	生成对应负向内部账本流水

⸻

十二、结算与提现规则

12.1 入账时机

订单支付成功后，立即生成所有收益明细和系统虚拟账户明细。

真实收益账户明细（含本地商家合作收益）初始状态为：待结算。

⸻

12.2 可提现时机

建议与399卡一致，配置为：

条件	说明
订单完成后可提现	可选
售后期结束后可提现	推荐
T+N天后可提现	可按财务规则配置

⸻

12.3 提现方式

本地商家的合作收益通过统一的余额分账能力提现，与共建/店家钱包提现流程一致。

⸻

十三、拉卡拉分账校验

与399卡一致：

当订单使用拉卡拉支付时，调用分账前校验：

拉卡拉可分账金额 <= 拉卡拉支付金额 - 商户主体预留金

商户主体预留金 = 拉卡拉支付金额 × 10%

若拉卡拉可分账额度不足，改走余额分账或待处理。

1. 本地商家有合作收益明细，收入名目为"合作收益"。
2. 共建/店家有推广佣金明细。
3. 平台营销费进入系统虚拟账户。
4. 技术服务费进入技术服务费虚拟账户。
5. 拉卡拉手续费不进入钱包账本。

十九、余额待分账池

本地商家业务所有分账出款（合作收益、各方推广佣金、技术服务费），统一通过平台余额待分账池处理。详见《余额待分账池》。

核心机制：

1. 用户下单支付后，拉卡拉自动扣除通道手续费，剩余金额全部进入余额待分账池；
2. 系统账本按订单原始金额足额记账，各方收益金额不受支付方式影响；
3. 实际打款来源均由余额待分账池承接，跨单滚动使用，不依赖单笔订单的拉卡拉可分账额度；
4. 积分、卡金、混合支付场景下的分账出款，同样由余额待分账池兜底；
5. 拉卡拉商户主体预留金（10%）留在收款商户，T+N天后释放补回资金池。

二十、最终产品口径总结

本地商家订单资金拆解：

订单金额 = 本地商家合作收益 + 平台毛利

平台毛利 = 技术服务费 + 商城推广佣金池

本地商家合作收益：
收入名目为"合作收益"，进入本地商家佣金钱包，走分销佣金体系统一结算。

商城推广佣金池分配：
直推（共建）30% + 间推（店家）30% + 平台营销费40%
缺失角色份额自动归入平台营销费账户。

技术服务费：
按订单全额1.5%计收，不受合作分成比例和支付方式影响。

统一钱包账本：
真实收益账户：本地商家钱包、共建钱包、店家钱包
系统虚拟账户：平台营销费账户、技术服务费账户

不进入统一钱包账本：
拉卡拉手续费、商户主体预留金

本地商家业务与399卡、980卡共用同一套"统一钱包账本 + 系统虚拟账户 + 支付通道规则解耦"架构，区别仅在于佣金池的计算方式不同（本地商家按平台毛利，399/980按固定10%）。
