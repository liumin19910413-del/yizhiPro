# 1万共建与30万联盟店身份购买需求方案

下面我按「1万共建」和「30万联盟店」重新梳理一版完整需求方案，重点覆盖：业务办理码、移动商家版办理、身份升级、合同签署、平台收益账本、奖金池、线下发放记录、收益统计。

---

## 一、整体方案定位

本次需求支持两类合伙人身份购买：

| 身份类型 | 金额 | 购买方式 | 合同方式 | 资金处理方式 | 身份激活方式 |
| --- | --- | --- | --- | --- | --- |
| 共建 | 1万元 | 线下POS/聚合码收款 + 移动商家版扫码办理 | 线上签约 | 平台收益账本入账 + 后续余额分账提现 + 收款商户留存 | 签约完成后激活 |
| 联盟店 | 30万元 | 线下POS/聚合码收款 + 移动商家版扫码办理 | 当前线下签约，系统预留上传 | 平台收益账本入账 + 后续余额分账提现 + 收款商户留存 | 办理完成后激活或按等级签约配置处理 |

核心原则：

线下POS/聚合码先收款，扣除手续费后的资金进入待分账资金池。员工使用移动商家版扫描对应业务办理码，搜索会员、确认上级关系、关联已支付的POS/聚合码支付单并提交办理。办理完成后，系统自动发放权益、升级合伙人身份、生成平台收益账本入账和水单；后续收益账户提现时，再通过余额分账流程从待分账资金池出款。

共建身份需要线上签约的，办理完成后生成合同签约二维码，客户完成签约后正式激活；联盟店当前可走线下签约，是否线上签约以合伙人等级配置为准。

本期将1万共建和30万联盟店归类为「市场明星业务」。市场明星业务不是普通推广商品或普通权益项目，而是平台为了促进市场启动、吸引关键合伙人进入体系而设置的特殊业务。它通过特殊的收益分配、奖励账本、总部保本/总部成本归集、销售收益规则和合同签约规则，先把共建、联盟店等关键市场角色建立起来，再带动后续普通推广项目、商品、套餐和服务的销售。

## 三、业务办理码与支付单关联

### 1. 办理入口

工作人员在移动商家版操作：

1. 扫描业务办理码；
2. 系统自动识别业务和套餐包；
3. 搜索客户资料；
4. 确认会员上级关系、销售归属关系；
5. 关联POS/聚合码已支付记录；
6. 提交办理；
7. 系统生成市场明星业务办理单。

业务办理码包括：

| 业务名称 | 金额 | 对应身份 | 对应权益包 |
| --- | --- | --- | --- |
| 1万共建 | 10000元 | 共建 | 1万共建权益包 |
| 30万联盟店 | 300000元 | 联盟店 | 30万联盟店权益包 |

---

### 2. 支付单关联字段

办理时需要关联POS/聚合码已支付记录，建议字段如下：

| 字段 | 说明 |
| --- | --- |
| 支付单类型 | POS / 聚合码 |
| 支付单号 | 线下收款交易号 |
| 支付金额 | 应与业务金额一致 |
| 支付时间 | 用于财务核对 |
| 支付方式 | 非信用卡 / 信用卡 / 聚合码支付方式 |
| 手续费率 | 按支付方式和通道规则识别 |
| 手续费金额 | 系统自动计算或从支付单带出 |
| 收款商户 | 实际收款主体 |
| 关联状态 | 未关联 / 已关联 / 异常 |

---

### 3. 金额校验规则

| 校验项 | 规则 |
| --- | --- |
| 金额校验 | 支付金额应与业务金额一致 |
| 流水重复校验 | 同一支付单不可重复关联多笔办理单 |
| 支付方式校验 | 必须识别非信用卡/信用卡，用于计算手续费 |
| 办理提交条件 | 未关联有效支付单，不允许提交办理 |
| 上级关系校验 | 员工必须确认会员上级关系、销售归属关系正确 |
| 异常处理 | 金额不一致、流水重复、支付方式缺失需提示异常 |

## 六、1万共建资金清分方案

### 1. 资金拆分规则

1万共建市场明星业务办理采用：

平台收益账本入账 + 后续余额分账提现 + 收款商户留存

| 资金项 | 比例 | 金额 | 处理方式 |
| --- | --- | --- | --- |
| 共建奖金池 | 10% | 1000元 | 进入平台收益账本的共建奖金池收益账户 |
| 总部成本账户 | 39% | 3900元 | 进入平台收益账本的总部保本/总部成本收益账户 |
| 销售收益 | 40% | 4000元 | 进入平台收益账本的销售相关合伙人收益账户 |
| 收款商户留存 | 剩余约11% | 1100元 | 留在收款商户侧 |
| POS手续费 | 非信用卡5‰ / 信用卡8‰ | 50元 / 80元 | 收款商户承担 |

---

### 2. 1万共建资金流程

```text
POS收款10000元
↓
员工扫描1万共建业务办理码
↓
搜索会员并确认上级关系
↓
关联POS/聚合码支付单
↓
提交市场明星业务办理
↓
生成平台收益账本入账：
  - 1000元 → 共建奖金池收益账户
  - 3900元 → 总部保本/总部成本收益账户
  - 4000元 → 按销售规则进入合伙人收益账户
↓
剩余1100元 → 收款商户留存
↓
系统发放1万共建权益包，升级共建身份，按签约配置生成合同二维码
↓
POS手续费由收款商户承担
  - 非信用卡：扣50元，收款商户实际约1050元
  - 信用卡：扣80元，收款商户实际约1020元
```

---

### 3. 1万共建规则说明

PRD中要明确：

共建奖金池、总部成本账户、销售收益均以业务原始金额10000元作为计算基数。POS手续费由收款商户承担，不影响奖金池、总部成本账户和销售收益金额。

## 八、平台收益账本与余额分账边界

这里必须在系统设计里拆清楚。

### 1. 平台收益账本负责

平台收益账本负责记录办理完成后各账户应得收益，不代表已经真实出款。

| 身份 | 收益账户 | 比例 |
| --- | --- | --- |
| 共建 | 共建奖金池账户 | 10% |
| 共建 | 总部成本账户 | 39% |
| 联盟店 | 联盟店奖金池账户 | 20% |
| 联盟店 | 总部成本账户 | 45% |

---

### 2. 销售收益规则负责

销售收益规则负责根据销售关系、上级关系、门店关系计算销售相关收益。

| 身份 | 收益对象 | 比例 |
| --- | --- | --- |
| 共建 | 销售/推广方 | 40% |
| 联盟店 | 销售门店/销售店家 | 24% |

销售收益进入对应合伙人或门店的收益账户，后续按收益提现流程打款。

```text
市场明星业务办理完成
↓
触发销售收益规则
↓
根据销售关系计算收益
↓
收益进入对应人员收益账户
↓
用户发起提现
↓
收益账户扣减
↓
通过余额分账打款到银行卡、支付宝或合规收款账户
```

---

### 3. 余额分账负责

余额分账负责收益账户提现时的真实出款。

1. POS/聚合码收款后，扣除手续费的资金进入待分账资金池；
2. 办理完成后，平台收益账本记录各方应得收益；
3. 收益账户提现时，从待分账资金池发起余额分账出款；
4. 钱包/收益账户入账和真实出款是两个流程。

---

### 4. 收款商户负责

收款商户承担：

1. 办理业务后的剩余留存；
2. POS手续费；
3. 30万联盟店可能存在的预留金；
4. 后续财务核对。

### 3. 奖金池账本余额与收支明细

奖金池需要支持查看账本余额和完整收支明细。

这里的奖金池余额指系统账本余额，不代表银行卡、支付宝或其他真实资金账户里的实时余额。它用于记录奖金池在系统中累计进来多少钱、支出去多少钱、当前账面还剩多少钱。

余额计算规则：

```text
奖金池账本余额 = 历史入账总额 - 历史支出总额
```

例如：

```text
共建奖金池账本余额10000元
线下发放支出2000元
支出后账本余额 = 8000元
```

收支明细字段建议：

| 字段 | 说明 |
| --- | --- |
| 明细编号 | 系统自动生成 |
| 奖金池类型 | 共建奖金池 / 联盟店奖金池 |
| 变动方向 | 入账 / 支出 / 冲正 |
| 变动金额 | 本次增加或减少的金额 |
| 变动前账本余额 | 本次变动前奖金池账本余额 |
| 变动后账本余额 | 本次变动后奖金池账本余额 |
| 来源类型 | 市场明星业务入账 / 线下奖金发放 / 人工调整 / 冲正 |
| 来源单号 | 办理单号 / 发放批次号 / 调整单号 |
| 关联对象 | 会员、合伙人、发放对象或操作对象 |
| 变动时间 | 系统记录时间 |
| 操作人 | 系统 / 后台操作人 |
| 备注 | 特殊说明 |

规则：

1. 市场明星业务办理完成后，奖金池生成入账明细，奖金池账本余额增加；
2. 线下奖金发放批次确认后，奖金池生成支出明细，奖金池账本余额减少；
3. 支出金额不得大于当前可用账本余额；
4. 每一笔余额变动都必须保留收支明细，不允许只改余额不留流水；
5. 历史收支明细需要支持按奖金池类型、时间、变动方向、来源类型查询。
6. 奖金池账本余额与真实收款账户、银行卡、支付宝等资金账户余额不要求实时一致，真实出款以线下发放或提现流程为准。

## 十三、关键验收标准

### 1. 共建市场明星业务办理验收

1. POS/聚合码收款10000元后，员工可在移动商家版扫描1万共建业务办理码；
2. 系统自动带出1万共建业务、10000元金额和1万共建权益包；
3. 员工可搜索会员，并必须确认上级关系、销售归属关系正确；
4. 办理单必须能关联已支付且未被占用的POS/聚合码支付单；
5. 提交办理后，系统生成共建市场明星业务办理单；
6. 系统按业务金额计算并生成平台收益账本入账：
    * 1000元进入共建奖金池收益账户；
    * 3900元进入总部保本/总部成本收益账户；
    * 4000元进入销售相关合伙人收益账户；
    * 剩余约1100元留给收款商户；
7. POS手续费由收款商户承担；
8. 系统自动发放1万共建权益包；
9. 系统自动升级会员为共建身份；
10. 如果共建等级配置为线上签约，办理结果页生成合同二维码；
11. 客户扫码后通过手机号验证码和勾选确认完成签约，共建身份变为已激活；
12. 系统调用现有接口生成水单，水单标识为业务来源 = 市场明星业务。

---

### 2. 联盟店市场明星业务办理验收

1. POS/聚合码收款300000元后，员工可在移动商家版扫描30万联盟店业务办理码；
2. 系统自动带出30万联盟店业务、300000元金额和30万联盟店权益包；
3. 员工可搜索会员，并必须确认上级关系、销售归属关系正确；
4. 办理单必须能关联已支付且未被占用的POS/聚合码支付单；
5. 提交办理后，系统生成联盟店市场明星业务办理单；
6. 系统按业务金额计算并生成平台收益账本入账：
    * 60000元进入联盟店奖金池收益账户；
    * 135000元进入总部保本/总部成本收益账户；
    * 72000元进入销售门店/销售店家收益账户；
    * 剩余约33000元留给收款商户；
7. POS手续费和预留金由收款商户承担；
8. 系统自动发放30万联盟店权益包；
9. 系统自动升级会员为联盟店身份，并按合伙人等级签约配置处理激活状态；
10. 若联盟店配置为线下签约，本期不展示线上签约二维码；
11. 系统调用现有接口生成水单，水单标识为业务来源 = 市场明星业务。

---

### 3. 奖金池验收

1. 共建市场明星业务办理完成后，共建奖金池自动生成1000元入账记录；
2. 联盟店市场明星业务办理完成后，联盟店奖金池自动生成60000元入账记录；
3. 奖金池入账后，奖金池账本余额自动增加；
4. 每笔入账都必须生成奖金池收支明细，变动方向为入账；
5. 后台支持按奖金池类型、时间、变动方向、来源类型查询收支明细；
6. 后台支持导入线下奖金发放明细；
7. 发放批次确认后，计入合伙人线下奖金；
8. 发放批次确认后，奖金池账本余额自动扣减；
9. 发放批次确认后，必须生成奖金池收支明细，变动方向为支出；
10. 支出金额不得大于奖金池当前可用账本余额；
11. 奖金池入账或支出后，可按配置复用已有微信推送能力提醒对应人群；
12. 合伙人总收益 = 线上收益 + 线下奖金。

## 十四、一期 MVP 定位

### 1. 本期目标

本期只优先支持「1万共建」和「30万联盟店」两个高优先级市场明星业务。

本期目标不是重做原收银、水单、提现系统，也不是一次性重构所有业务线，而是在现有能力基础上新增一套轻量的市场明星业务办理链路，完成：

1. 市场明星业务配置；
2. 业务办理码；
3. 移动商家版扫码办理；
4. POS/聚合码支付单关联；
5. 平台收益账本入账；
6. 自动生成水单；
7. 自动发放权益；
8. 自动升级合伙人身份；
9. 按合伙人等级规则生成合同签约入口；
10. 后续收益提现继续复用现有余额分账出款能力。

### 2. 本期不包含范围

以下业务不纳入第一期 MVP：

1. 399控制糖卡；
2. 980团购卡；
3. 本地商家合作收益；
4. 第三方供应链业务；
5. 新建水单页面；
6. 重做提现系统；
7. 复杂自动冲正、自动追回流程；
8. 第三方电子签接入。

### 3. 设计原则

本期按最小可用原则实现：

1. 业务办理入口独立，减少与原普通收银流程的强耦合；
2. 会员、上级关系、合伙人等级、套餐权益、合同模板、收益提现等已有能力优先复用；
3. 办理完成后先进入平台收益账本，不直接等同于真实出款；
4. 收益账户提现时，再通过余额分账流程从待分账资金池出款；
5. 水单复用现有页面和接口，只增加业务来源标识。

## 十五、平台收益账本

### 1. 定位

平台收益账本是平台用于记录各类业务收益、系统收益账户、合伙人收益账户和收益流水的结算账本。

平台收益账本不定义为支付账户、储值账户或可自由支付的钱包。它只用于记录业务收益归属、提现前余额、入账明细和后续结算依据。

真实出款仍通过现有余额分账提现流程完成。

### 2. 收益名目管理

收益名目用于说明一笔钱为什么产生，以及对外展示时应该叫什么。

| 字段 | 说明 |
| --- | --- |
| 名称 | 后台配置名称，例如自推奖、间推奖、总部保本、共建奖金池 |
| 收益名目 | 对外展示名目，例如自推奖、间推奖、自购奖、总部保本、奖金池 |
| 备注说明 | 说明该收益产生原因、适用业务和使用边界 |
| 来源类型 | 系统内置 / 自定义 |
| 状态 | 启用 / 停用 |

操作规则：

1. 系统内置收益名目不可删除；
2. 系统内置收益名目可编辑备注说明，可启用或停用；
3. 自定义收益名目可新增、编辑、删除；
4. 已被业务配置或历史流水引用的自定义收益名目，不建议物理删除，应改为停用；
5. 停用后不允许新业务继续选择，但历史流水仍正常展示。

本期建议内置收益名目：

| 名称 | 收益名目 | 说明 |
| --- | --- | --- |
| 自推奖 | 自推奖 | 直属销售或本人推广产生的收益 |
| 间推奖 | 间推奖 | 上级关系或间接推广产生的收益 |
| 自购奖 | 自购奖 | 自购场景产生的收益 |
| 总部保本 | 总部保本 | 总部侧保本或成本归集收益 |
| 共建奖金池 | 奖金池 | 1万共建业务进入共建奖金池的收益 |
| 联盟店奖金池 | 奖金池 | 30万联盟店业务进入联盟店奖金池的收益 |

### 3. 系统收益账户管理

系统收益账户用于承接总部保本、奖金池等平台内部或专项收益。

| 字段 | 说明 |
| --- | --- |
| 账户名称 | 例如总部保本账户、共建奖金池账户、联盟店奖金池账户 |
| 绑定收益名目 | 从收益名目管理中选择 |
| 收款账户 | 用于后续提现或自动提现的银行卡、支付宝或其他合规收款账户 |
| 是否自动提现 | 是 / 否 |

| 状态 | 启用 / 停用 |

规则：

1. 市场明星业务配置中只选择系统收益账户，不在业务配置里重复维护收款信息；
2. 系统收益账户入账后，是否自动提现由该账户自身配置决定；
3. 自动提现仍走现有余额分账提现流程；
4. 本期不重做提现系统。

## 十六、市场明星业务配置与办理码

### 1. 市场明星业务配置

新增轻量的市场明星业务配置，用于配置「1万共建」和「30万联盟店」这类特殊业务。

市场明星业务的核心目的，是通过特殊分成机制促进市场动起来。它区别于推广项目中已有的普通商品、套餐、服务权益：普通推广项目主要解决“推什么项目”，市场明星业务主要解决“先让谁成为市场关键角色，并按特殊规则分配收益和奖励”。

| 字段 | 说明 |
| --- | --- |
| 业务名称 | 例如1万共建、30万联盟店 |
| 业务金额 | 10000元 / 300000元 |
| 绑定合伙人等级 | 办理成功后升级到的合伙人等级 |
| 关联权益套餐 | 办理成功后自动发放的权益套餐，例如1万共建权益包、30万联盟店权益包 |
| 总部保本账户 | 从平台收益账本的系统收益账户中选择 |
| 总部保本金额/比例 | 支持按固定金额或比例配置 |
| 奖金池账户 | 从平台收益账本的系统收益账户中选择 |
| 奖金池金额/比例 | 支持按固定金额或比例配置 |
| 销售收益规则 | 复用现有销售收益、上级关系或门店分润规则 |
| 是否生成办理码 | 是 / 否 |
| 状态 | 启用 / 停用 |

配置边界：

1. 总部保本/总部成本比例、奖励池比例、销售收益比例都在「市场明星业务配置」中维护；
2. 1万共建的10%进入共建奖励池、30万联盟店的20%进入联盟店奖励池，属于具体明星业务的分配规则，应跟业务金额、绑定身份、关联权益套餐放在同一处配置；
3. 「奖励设置」不重复配置入账比例，只配置奖励账户、是否推送、推送对象、发放周期等通用规则；
4. 同一业务的分配比例只能有一个生效来源，避免市场明星业务和奖励设置两边重复配置导致口径冲突。
5. 市场明星业务必须关联权益套餐；未关联权益套餐时，不允许生成办理码或提交办理。

本期默认配置：

| 业务 | 金额 | 绑定身份 | 关联权益套餐 | 总部保本 | 奖励池 | 销售收益 |
| --- | --- | --- | --- | --- | --- | --- |
| 1万共建 | 10000元 | 共建 | 1万共建权益包 | 3900元或39% | 1000元或10% | 复用现有规则 |
| 30万联盟店 | 300000元 | 联盟店 | 30万联盟店权益包 | 135000元或45% | 60000元或20% | 复用现有规则 |

### 2. 业务办理码

每个启用的市场明星业务可生成一个业务办理码。

业务办理码不是支付码，只用于让员工在移动商家版中进入指定业务办理流程。

规则：

1. 员工必须先登录移动商家版；
2. 员工扫码后，系统识别对应业务；
3. 如果扫码的是1万共建业务码，则自动带出1万共建业务和对应套餐包；
4. 如果扫码的是30万联盟店业务码，则自动带出30万联盟店业务和对应套餐包；
5. 停用的业务办理码不可继续办理。

## 十九、余额待分账池

1万共建和30万联盟店的实际出款，统一通过平台余额待分账池处理。详见《余额待分账池》。

核心机制：

1. 线下POS/聚合码收款场景：收款资金扣除手续费后，进入余额待分账池；
2. 员工办理市场明星业务并关联支付单后，系统按业务金额生成平台收益账本入账；
3. 平台收益账本记录各账户应得收益，不等同于立即真实出款；
4. 收益账户后续提现时，再通过余额分账流程从待分账资金池出款；
5. 系统账本按业务原始金额足额记账，收益金额不受POS手续费影响；
6. POS手续费（非信用卡5‰/信用卡8‰）由收款商户承担，不影响总部保本、奖金池和销售收益金额计算。

## 二十、最终一句话总结

这套方案可以概括为：

1万共建和30万联盟店通过线下POS/聚合码先收款、员工移动商家版扫码办理、关联支付单完成业务闭环。办理完成后，系统自动发放权益、升级合伙人身份、生成平台收益账本入账和水单；收益账户后续提现时，再复用余额分账流程从待分账资金池出款。共建等需要线上签约的身份，在办理完成页生成合同二维码，客户通过手机号验证码和勾选确认完成签约后正式激活。
