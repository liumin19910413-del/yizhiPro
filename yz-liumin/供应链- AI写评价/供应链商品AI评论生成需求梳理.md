# 供应链评论管理与 AI 写评论需求文档

## 1. 文档定位

本文档用于研发开发和测试验收，描述评论管理页面与 `AI写评论` 功能的页面范围、权限边界、字段口径、交互规则、数据规则和验收标准。

本期以当前交互演示稿为准，研发实现时不再按“演示理解”二次推断。

## 2. 业务背景

供应链商品会同步到商家线上商城售卖，消费者完成购买后会产生真实评论。部分商品评论量较少，影响商品转化，因此平台需要提供 AI 写评论能力，为评论较少的商品补充基础评论内容。

本期同时覆盖两类后台使用场景：

1. 供应链后台：统一查看全部评论，并可发起 `AI写评论`。
2. 商家后台：查看自身可见范围内的评论，不可发起 `AI写评论`。

## 3. 本期目标

### 3.1 评论管理

1. 支持按后台身份进入评论管理页面。
2. 列表统一展示 `真实评论` 和 `AI评论`。
3. 评论来源、评论时间、星级、归属门店/所属门店、商品信息、会员信息等信息清晰可见。
4. 支持按筛选条件快速定位评论数据。

### 3.2 AI 写评论

1. 仅供应链后台可见 `AI写评论` 入口。
2. 支持对全部商品发起 AI 写评论任务。
3. 支持对“评论少于指定条数”的商品发起 AI 写评论任务。
4. 支持配置每个商品随机生成的评论条数区间。

## 4. 页面范围与角色权限

### 4.1 页面入口

统一入口：`电商供应链 > 评论管理`

### 4.2 视角说明

本期页面存在 3 个业务视角：

1. `供应链后台`
2. `商家后台-总部后台`
3. `商家后台-分店后台`

### 4.3 权限矩阵

| 能力项 | 供应链后台 | 商家后台-总部后台 | 商家后台-分店后台 |
|---|---|---|---|
| 查看真实评论 | 支持，查看全部门店 | 支持，仅查看当前商家体系下门店 | 支持，仅查看当前分店 |
| 查看 AI 评论 | 支持 | 支持 | 支持 |
| 查看门店列 | 支持，字段名为`归属门店` | 支持，字段名为`所属门店` | 不支持 |
| 门店筛选 | 支持，全部门店可选 | 支持，仅可选当前商家体系下门店 | 不支持 |
| `AI写评论` 入口 | 支持 | 不支持 | 不支持 |

## 5. 页面结构

评论管理页由以下区域组成：

1. 页面标题区
2. 视角切换区
3. 筛选区
4. 评论列表区
5. 评论详情抽屉
6. `AI写评论` 弹窗

## 6. 评论管理页面需求

### 6.1 筛选区展示顺序

筛选项展示顺序固定如下：

1. 评论时间
2. 评论来源
3. 星级
4. 归属门店 / 所属门店
5. 关键字搜索
6. 查询
7. 清空筛选
8. `AI写评论`

说明：

1. `归属门店 / 所属门店` 仅在供应链后台、商家后台-总部后台显示。
2. `AI写评论` 按钮仅在供应链后台显示。

### 6.2 筛选项定义

| 筛选项 | 控件类型 | 说明 |
|---|---|---|
| 评论时间 | 下拉 | 可选：`全部时间`、`今天`、`近7天`、`近30天` |
| 评论来源 | 下拉 | 可选：`全部`、`真实评论`、`AI评论` |
| 星级 | 下拉 | 可选：`全部星级`、`5星`、`4星`、`3星`、`2星`、`1星` |
| 归属门店 / 所属门店 | 下拉 | 供应链后台可选全部门店；商家总部仅可选当前商家体系下门店 |
| 关键字搜索 | 输入框 | 支持按`商品名称`、`会员昵称`、`手机号`模糊搜索 |

### 6.3 关键字搜索规则

1. 关键字搜索为单输入框。
2. 支持同时命中以下字段：
   1. 商品名称
   2. 会员昵称
   3. 手机号
3. 搜索方式为模糊匹配，满足任一字段命中即可返回。
4. 手机号建议按原始手机号字段查询，页面展示仍使用脱敏手机号。
5. 当关键字为空时，不作为筛选条件。

### 6.4 查询与重置规则

1. 点击 `查询` 后，按当前全部筛选条件刷新列表。
2. 点击 `清空筛选` 后，所有筛选项恢复默认值，再刷新列表。
3. 默认值如下：
   1. 评论时间：`全部时间`
   2. 评论来源：`全部`
   3. 星级：`全部星级`
   4. 门店：`全部门店`
   5. 关键字搜索：空

### 6.5 列表字段定义

#### 6.5.1 供应链后台列表字段

| 列名 | 说明 |
|---|---|
| 评论时间 | 评论提交时间或 AI 评论生成时间，精确到分钟 |
| 会员信息 | 第一行显示会员昵称，第二行显示脱敏手机号 |
| 评论来源 | `真实评论` / `AI评论` |
| 商品信息 | 商品名称 |
| 评论内容 | 展示星级和评论正文 |
| 关联订单号 | 真实评论展示实际订单号；AI 评论展示 `-` |
| 归属门店 | 真实评论展示实际门店；AI 评论展示 `门店通用` |

#### 6.5.2 商家后台-总部后台列表字段

| 列名 | 说明 |
|---|---|
| 评论时间 | 同供应链后台 |
| 会员信息 | 同供应链后台 |
| 评论来源 | 同供应链后台 |
| 商品信息 | 同供应链后台 |
| 评论内容 | 同供应链后台 |
| 关联订单号 | 同供应链后台 |
| 所属门店 | 真实评论展示门店；AI 评论展示 `门店通用` |

#### 6.5.3 商家后台-分店后台列表字段

| 列名 | 说明 |
|---|---|
| 评论时间 | 同供应链后台 |
| 会员信息 | 同供应链后台 |
| 评论来源 | 同供应链后台 |
| 商品信息 | 同供应链后台 |
| 评论内容 | 同供应链后台 |
| 关联订单号 | 同供应链后台 |

### 6.6 列表展示规则

1. 列表必须同时展示 `真实评论` 和 `AI评论`。
2. `真实评论` 必须带实际门店信息。
3. `AI评论` 在存在门店列的视角下统一显示为 `门店通用`。
4. `AI评论` 的会员昵称统一显示为 `AI生成`。
5. `AI评论` 无真实订单时，关联订单号显示 `-`。
6. 评论内容区需要展示星级和正文。
7. 评论来源需使用明显标识区分 `真实评论` 与 `AI评论`。

### 6.7 各视角数据范围规则

#### 6.7.1 供应链后台

1. 可查看全部真实评论。
2. 可查看全部 AI 评论。
3. 门店筛选项展示全部门店。
4. 当选择某门店时，仅筛选真实评论的门店归属；AI 评论仍然可见。

#### 6.7.2 商家后台-总部后台

1. 可查看当前商家体系下所有门店的真实评论。
2. 可查看门店通用的 AI 评论。
3. 门店筛选项仅展示当前商家体系下的门店。
4. 当选择某门店时，仅筛选真实评论的门店归属；AI 评论仍然可见。

#### 6.7.3 商家后台-分店后台

1. 仅可查看当前分店的真实评论。
2. 可查看门店通用的 AI 评论。
3. 不显示门店筛选。
4. 不显示门店列。

### 6.8 评论详情抽屉

点击列表 `查看` 后，打开评论详情抽屉。

#### 6.8.1 字段定义

| 字段 | 说明 |
|---|---|
| 归属门店 / 所属门店 | 仅在供应链后台、商家总部展示；AI 评论显示 `门店通用` |
| 商品名称 | 评论对应商品 |
| 会员昵称 | 真实评论显示真实会员昵称；AI 评论显示 `AI生成` |
| 手机号 | 展示脱敏手机号；AI 评论可展示 `-` |
| 评论来源 | `真实评论` / `AI评论` |
| 星级 | 1-5 星 |
| 评论时间 | 评论提交时间或生成时间 |
| 评论内容 | 评论正文 |
| 关联订单号 | 真实评论展示订单号；AI 评论展示 `-` |

#### 6.8.2 展示规则

1. 商家后台-分店后台不展示门店字段。
2. AI 评论字段展示口径需与列表一致。

## 7. AI 写评论需求

### 7.1 入口规则

1. 仅供应链后台展示 `AI写评论` 按钮。
2. 商家后台任一视角均不展示该按钮。

### 7.2 弹窗结构

弹窗包含以下区域：

1. 标题区
2. 基础配置区
3. 提交操作区
4. 提交结果区
5. 消费者端效果预览区

### 7.3 基础配置项

| 配置项 | 类型 | 说明 |
|---|---|---|
| 写评论范围 | 单选 | `全部商品` / `评论少于指定条数的商品` |
| 每个商品写评论条数 | 单选 | `3-5条` / `3-10条` / `3-20条` |
| 评论少于多少条 | 数字输入 | 仅在选择 `评论少于指定条数的商品` 时显示 |
| 符合条件商品数 | 只读展示 | 实时展示当前命中的商品数量 |

### 7.4 写评论范围规则

#### 7.4.1 全部商品

1. 对全部上架商品发起写评论任务。
2. 右侧预览展示一个示例商品的消费者端评论样式。

#### 7.4.2 评论少于指定条数的商品

1. 用户输入阈值，例如 `3`。
2. 系统实时筛出“当前评论数 < 3”的商品。
3. 所有符合条件商品都进入本次任务范围。
4. 右侧预览可从符合条件商品中任选一个商品作为预览样本。
5. 若符合条件商品数为 `0`，则不允许提交。

### 7.5 提交规则

点击 `确认刷评论` 后，按以下规则处理：

1. 当范围为 `全部商品` 时，允许直接提交。
2. 当范围为 `评论少于指定条数的商品` 时：
   1. 若无符合条件商品，不允许提交。
   2. 需提示：`当前筛选条件下暂无符合条件商品。`
3. 提交成功后，展示明确结果文案。

### 7.6 提交结果文案

#### 7.6.1 全部商品

示例：

`已提交：对全部上架商品刷 3-5 条评论。`

#### 7.6.2 评论少于指定条数的商品

示例：

`已提交：对评论少于 3 条的商品随机刷 3-5 条评论。`

如需增强可读性，可额外展示本次命中的商品名称列表。

### 7.7 消费者端效果预览

1. 预览仅用于展示 AI 评论在消费者端的样式效果。
2. 预览不代表实际落库结果。
3. 预览商品在“全部商品”模式下可固定为默认示例商品。
4. 预览商品在“评论少于指定条数”模式下应从命中商品中选择。

## 8. 数据与口径约定

### 8.1 评论来源枚举

建议后端统一枚举值：

| 枚举值 | 含义 |
|---|---|
| `real` | 真实评论 |
| `ai` | AI评论 |

### 8.2 评论数据字段建议

评论列表返回建议至少包含以下字段：

| 字段 | 说明 |
|---|---|
| commentId | 评论 ID |
| sourceType | 评论来源：`real` / `ai` |
| commentTime | 评论时间 |
| rating | 星级 |
| content | 评论正文 |
| productId | 商品 ID |
| productName | 商品名称 |
| memberId | 会员 ID，AI 评论可为空 |
| memberName | 会员昵称 |
| memberPhone | 原始手机号，AI 评论可为空 |
| memberPhoneMasked | 脱敏手机号 |
| orderNo | 关联订单号，AI 评论可为空 |
| storeId | 真实评论必填，AI 评论可为空 |
| storeName | 门店名称，AI 评论可为空 |

### 8.3 AI 评论展示口径

1. 当 `sourceType = ai` 时：
   1. 会员昵称展示 `AI生成`
   2. 门店列展示 `门店通用`
   3. 订单号展示 `-`
   4. 手机号展示 `-`
2. 当 `sourceType = real` 时，按真实业务数据展示。

### 8.4 时间筛选口径

1. `今天`：自然日内数据。
2. `近7天`：含当天，向前滚动 7 天。
3. `近30天`：含当天，向前滚动 30 天。

### 8.5 星级筛选口径

按评论星级精确匹配，取值范围 1-5。

## 9. 前后端实现要求

### 9.1 列表查询

列表查询接口需支持以下入参：

1. 当前视角
2. 当前用户可见门店范围
3. 评论时间范围
4. 评论来源
5. 星级
6. 门店
7. 关键字

### 9.2 权限控制

1. 前端需按角色控制按钮和字段显隐。
2. 后端也必须做数据权限兜底，不能只依赖前端隐藏。
3. 商家后台不得通过改参方式发起 `AI写评论`。

### 9.3 AI 写评论任务提交

提交接口需至少支持以下入参：

1. scope：`all` / `lessThan`
2. threshold：当 `scope = lessThan` 时必填
3. countRange：`3-5` / `3-10` / `3-20`

后端需校验：

1. 当前用户是否有发起权限
2. `threshold` 是否合法
3. 当 `scope = lessThan` 时是否存在符合条件商品

| TC-14 | 商家总部按门店筛选 | 选择某门店查询 | 真实评论按门店筛选，AI 评论仍可见 |
| TC-15 | 清空筛选 | 选择多个条件后点击 `清空筛选` | 所有条件恢复默认值并刷新列表 |
| TC-16 | 供应链后台显示 `AI写评论` | 进入供应链后台 | 页面展示 `AI写评论` 按钮 |

## 13. 开发注意事项

1. 权限控制需要前后端双重校验。
2. 商家后台的数据权限必须基于登录主体限制，不允许仅靠前端筛选控制。
3. 手机号搜索建议对接原始手机号字段，避免脱敏串无法稳定匹配。
4. “评论少于指定条数”的商品筛选必须基于实时评论总数计算。
5. 列表、详情、AI 评论展示口径必须保持一致，避免同一条评论在不同位置展示不同文案。
