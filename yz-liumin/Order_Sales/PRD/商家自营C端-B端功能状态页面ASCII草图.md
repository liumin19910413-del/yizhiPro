# 商家自营 C 端 / B 端功能状态页面 ASCII 草图

> 依据：`商家自营订单售后C-B联动完整PRD.md`、`C端参考图/`、`B端参考图/`、`设计规范/设计规范.md`。
>
> 草图原则：C 端贴近参考图的小程序页面结构，保留浅柔状态头、白色圆角卡片、玫瑰粉品牌金额与底部固定主按钮；B 端贴近参考图的后台工作台结构，保留左侧导航、顶部筛选、状态 Tab、表格、右侧详情窗口和中央动作弹窗。
>
> 注意：以下不是流程说明页，而是后续 Demo 可落地的真实产品页面骨架。

---

## 0. 设计规范对照

本草图按 `awesome-design-md` 的参考方向落地：B 端参考 Linear / Vercel / Stripe 的工具型产品气质，C 端参考 Apple / Notion / Figma / Shopify 的消费级产品气质，并以本项目 `设计规范/设计规范.md` 为优先口径。

| 规范项 | 草图落地方式 |
| --- | --- |
| B 端清晰、稳定、专业、高密度 | 使用左侧导航、顶部筛选、状态 Tab、数据表格、右侧详情窗口，复杂处理进入中央弹窗 |
| B 端主画布近白或浅灰 | 后台草图以浅灰工作区 + 白色内容面板表达，不做营销式大图或装饰 |
| B 端状态命名与业务一致 | 使用 `待审核 / 待寄回 / 待收货 / 待退款 / 退款中 / 退款失败 / 待补寄出库 / 待买家收货 / 已完成 / 已关闭`，已拒绝和已撤销只作为关闭类型展示 |
| B 端金额、退款、库存操作需二次确认 | 审核、拒绝、质检、退款、重新退款、补寄出库都设计为中央弹窗确认 |
| C 端精致、温和、可信 | 使用小程序顶栏、浅柔状态头、圆角内容卡、清晰金额和底部固定主 CTA |
| C 端流程短且反馈明确 | 订单入口、售后类型、申请页、详情页、物流填写页按真实用户路径串联 |
| 双端状态一致 | 单独提供 C/B 状态联动表，保证消费者展示与后台处理节点一一对应 |
| 金额与权益口径一致 | C 端展示申请金额、审批金额、实际退回方式；B 端展示退款清分账单和资产回退 |
| 色彩原则 | C 端主 CTA 与价格建议使用品牌玫瑰粉 `#C65B7C`，成功用绿色、失败用危险色；B 端只在主操作、选中态、关键状态使用品牌色 |
| 圆角原则 | C 端卡片按 12-16px 柔和圆角；B 端表格、弹窗、卡片按 6-8px 克制圆角 |

状态差异：

| 场景 | 申请类型 | 原因枚举 | 运费提示 | 凭证 |
| --- | --- | --- | --- | --- |
| 待发货仅退款 | `仅退款` | 不想要了、拍错/多拍、地址错误、商家迟迟未发货等 | 全额含运费；部分默认不退运费 | 可不传 |

| 退货退款 | `退货退款（已收到货）` | 不想要了、规格不合适、质量问题、发错货、少件、描述不符等 | 默认不退原订单运费 | 质量/破损/错发等建议上传 |

### 1.5 非物流退款确认弹窗

适用：自提待自提、套餐/次卡未核销、储值资产满足回退、预定金权益未使用。

```text
+--------------------------------------------------+
|  订单详情                                        |
|  ...                                             |
|                         [申请退款]               |
+--------------------------------------------------+
|  灰色遮罩                                        |
|                                                  |
|      +------------------------------------+      |
|      |              退款申请              |      |
|      |                                    |      |
|      |  退款申请提交后，将等待商家处理退款 |      |
|      |  是否确认发出申请？                |      |
|      +----------------+-------------------+      |
|      |      取消      |      申请退款      |      |
|      +----------------+-------------------+      |
```

确认后：创建售后单，系统默认理由 `不想要了`，C 端进入 `待退款` 售后详情；不进入申请页，不展示原因、说明、凭证、商品数量选择。
