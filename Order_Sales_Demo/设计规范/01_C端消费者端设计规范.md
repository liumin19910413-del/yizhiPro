# C端消费者端设计规范

## 1. 设计目标

本规范用于指导 OpenDesign 生成「商家自营售后体系」C端消费者页面原型。

C端消费者页面重点表达：

1. 用户从订单详情进入退款 / 售后申请流程。
2. 用户根据订单状态选择售后类型。
3. 用户填写退款 / 售后申请信息。
4. 用户查看售后详情与处理进度。
5. 不同售后状态下展示不同结果页，例如退款成功、申请驳回、待商家处理、已撤销等。

整体风格应接近真实小程序 / App 页面，不要做成后台管理系统，不要做成三栏联动看板。

---

## 2. 整体视觉风格

### 2.1 页面气质

C端页面整体应呈现：

- 简洁、轻量、白底为主；
- 卡片化分区，信息层级清晰；
- 重点金额和操作按钮使用红橙色突出；
- 强业务状态表达，用户一眼知道当前售后状态；
- 页面应像真实消费者端订单 / 售后页面，而不是流程图组件或后台表格。

### 2.2 页面尺寸

按照移动端小程序页面设计。

建议画布尺寸：

```text
宽度：375px
高度：812px 或根据内容滚动延展
```

页面主体结构：

```text
顶部状态栏
导航栏
内容区
底部安全区 / 底部操作栏
```

## 10. OpenDesign 生成约束

生成 C端消费者端页面时，请严格遵守：

1. 不要生成三栏联动看板。
2. 不要把订单列表、订单详情、售后详情同时放在一个页面。
3. 每个页面必须是独立手机页面。
4. 每个流程节点用一个独立页面卡片展示。
5. 页面之间用箭头连接。
6. C端页面只展示消费者可见内容。
7. 不要出现 B端商家后台页面。
8. 页面视觉必须接近真实小程序 / App。
9. 订单详情、售后类型、申请退款、售后详情需要分别独立成页。
10. 售后状态必须清楚标识在页面顶部。
11. 金额必须用红色突出。
12. 表单页必须有固定底部主按钮。
13. 弹窗必须使用遮罩层和居中确认框。
14. 商品、订单、售后信息必须用卡片分组展示。
15. 如果一个场景包含多个状态，应画成流程面板，而不是合并到一个页面。
