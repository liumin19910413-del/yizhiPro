# 代客户下单的业务逻辑说明

## 1. 功能定位

代客户下单是指商户后台销售人员在商户侧代替客户创建一笔微商城订单，并将付款链接发送给客户，由客户后续自行完成支付。

本期范围明确限定为：

- `仅支持微商城订单`
- 不支持拼团订单、秒杀订单、积分订单、充值订单、供应链订单等其他订单类型

该能力的核心目标是：

- 降低客户下单门槛，提升成交转化率
- 支持销售在沟通场景下快速帮客户完成下单动作
- 保持订单仍由客户完成最终支付，确保交易链路与普通商城订单一致

该功能与“待付款订单改价”是两套独立能力：

- `代客户下单`：是创建一笔新订单，允许在提交前编辑商品、数量、地址、邮费、订单总价
- `待付款订单改价`：是对客户已提交但未支付的订单进行金额调整，不重建订单内容

## 2. 功能需求

### 2.1 入口需求

- 商户后台 `PC 端` 微商城订单列表增加 `代客户下单` 入口
- 商户后台 `移动端` 微商城订单列表增加 `代客户下单` 入口
- 点击入口后进入代客户下单流程
- 入口只在 `微商城订单` 页面增加，不扩展到其他订单类型页面

### 2.2 下单流程需求

销售发起代客户下单后，需要按以下顺序完成订单组装：

1. 选择会员
2. 选择微商城商品
3. 调整商品数量
4. 选择收货地址
5. 支持新建地址
6. 修改邮费，默认邮费为 `0`
7. 修改订单总价
8. 提交订单

### 2.3 会员选择需求

- 不默认带出会员，初始为空状态
- 支持按手机号、姓名等关键词搜索会员
- 选择会员后，自动带出该会员默认收货地址
- 支持重新选择会员

### 2.4 商品选择需求

- 仅允许选择 `微商城已上架商品`
- 支持按商品关键词搜索
- 支持加入购物车后调整数量
- 支持删除已选商品
- 商品总价随商品与数量实时联动计算

### 2.5 地址选择需求

- 默认填入客户默认收货地址
- 支持从客户地址列表中切换其他地址
- 支持为客户新建地址
- 新建地址后可立即选中作为本次订单地址

### 2.6 金额编辑需求

- 默认邮费为 `0`
- 销售可手动修改邮费
- 支持修改订单总价
- 订单总价修改后，应以修改后的最终金额作为本次订单应付金额
- 系统需要保留原始商品金额、邮费、调整后订单金额等字段，便于后续追溯

### 2.7 提交与支付需求

- 提交成功后，系统创建一笔新的待付款订单
- 系统自动复制付款链接到剪贴板
- 页面返回订单列表
- 新订单在商户后台订单列表中需带有 `代客户下单` 标识

### 2.8 待付款状态下的后台可操作按钮

代客户下单创建的订单，在 `待付款` 状态下，商户后台可操作按钮为：

- `编辑`
- `删单`
- `复制付款链接`

### 2.9 客户支付需求

- 客户通过销售发送的付款链接进入订单详情页
- 系统自动识别该订单商品是否支持 `会员卡余额支付`
- 若支持会员卡余额支付，则需要展示会员卡抵扣信息，并允许客户切换可用会员卡
- 若当前会员卡余额足够覆盖订单，则 `还需支付 = 0`，客户点击 `去付款` 可直接完成支付
- 若当前会员卡余额不足覆盖订单，则页面需展示 `某张卡已抵扣金额 + 还需微信支付金额`
- 若存在多张可用会员卡，则客户可切换不同会员卡查看抵扣结果
- 若没有可用会员卡余额，则订单详情页直接按普通待付款订单展示，点击 `去付款` 调起微信支付
- 客户可点击 `去付款` 完成支付
- 客户也可在小程序 `订单中心 - 待付款订单` 中查看该订单
- 客户支付成功后，订单流转为后续正常履约状态

## 3. 业务逻辑

### 3.1 业务本质

代客户下单本质上是商户侧代替客户完成“下单动作”，但不代替客户完成“支付动作”。

因此系统需要保证：

- 订单是新创建的正式订单
- 订单收货信息、商品信息、订单金额在提交时即确定
- 客户支付链路与普通商城订单一致

### 3.2 金额逻辑

代客户下单属于“订单尚未生成前的编辑”，因此允许在提交前修改订单金额。

建议金额口径为：

- `商品原始总价`：商品单价 × 数量汇总
- `邮费`：销售手动输入，默认 `0`
- `订单调整后总价`：销售确认后的最终订单总额

建议系统内部保留以下关系：

- 若未手动改总价：`最终应付金额 = 商品原始总价 + 邮费`
- 若手动改总价：`最终应付金额 = 手动修改后的订单总价`

同时需要记录：

- 原始商品金额
- 原始邮费
- 修改后订单总价
- 调整原因（建议保留，便于追溯）

### 3.3 订单状态逻辑

- 销售提交订单成功后：订单状态为 `待付款`
- 客户支付成功后：订单状态流转为后续正常状态，如 `待发货`

### 3.4 订单标识逻辑

- 商户后台订单列表显示 `代客户下单` 标识
- 客户端不展示该标识，客户视角与普通订单一致

### 3.5 权限逻辑

- 仅有对应权限的商户后台账号可发起代客户下单
- 销售只能对自己权限范围内的客户进行代下单

### 3.6 编辑逻辑

代客户下单产生的订单，在客户未支付前允许后台编辑，但编辑后应满足：

- 更新订单商品、数量、地址、邮费、最终金额
- 若付款链接仍沿用原链接，则链接对应的订单详情页需展示最新订单内容与最新金额
- 若业务上采用重新生成支付信息，则需保证客户只能按最新版本订单支付

### 3.7 删单逻辑

- 仅待付款状态允许删单
- 删单需二次确认
- 删单后原付款链接失效

### 3.8 会员卡支付逻辑

- 仅当该代客户下单订单中的商品满足会员卡支付规则时，客户侧订单详情页才展示会员卡抵扣模块
- 会员卡可用性判断在客户打开付款链接或进入待付款订单详情页时实时执行
- 若存在多张可用会员卡，默认按系统推荐卡展示，同时支持客户主动切换
- 切换会员卡后，页面需实时刷新：
  - 本次会员卡抵扣金额
  - 剩余需微信支付金额
  - 最终支付方式说明
- 若会员卡余额足额：
  - 还需支付显示为 `0`
  - 点击 `去付款` 后直接完成支付，不再调起微信支付
- 若会员卡余额不足：
  - 先展示会员卡已抵扣金额
  - 再展示剩余应支付金额
  - 点击 `去付款` 后调起微信支付完成差额支付
- 若不可用会员卡或无余额：
  - 不展示卡余额抵扣区
  - 页面行为与普通待付款订单一致，直接微信支付

## 4. 业务流程

### 4.1 主流程

```text
商户后台订单列表点击代客户下单
→ 搜索并选择会员
→ 系统自动带出默认收货地址
→ 搜索并选择微商城商品
→ 调整商品数量
→ 选择或新建收货地址
→ 设置邮费
→ 视需要修改订单总价
→ 提交订单
→ 系统创建待付款订单
→ 自动复制付款链接
→ 返回订单列表并展示代客户下单标识
→ 客户打开付款链接或在订单中心查看待付款订单
→ 客户完成支付
→ 订单进入后续履约状态
```

### 4.2 异常流程

#### 4.2.1 未选择会员

```text
销售点击提交
→ 系统校验未选择会员
→ 阻止提交并提示先选择会员
```

#### 4.2.2 未选择商品

```text
销售点击提交
→ 系统校验购物车为空
→ 阻止提交并提示先选择商品
```

#### 4.2.3 未选择地址

```text
销售点击提交
→ 系统校验未填写收货地址
→ 阻止提交并提示补全地址
```

#### 4.2.4 库存不足

```text
销售提交订单
→ 系统校验库存不足
→ 阻止提交并提示调整商品数量
```

#### 4.2.5 付款链接失效

```text
客户打开付款链接
→ 系统识别订单已关闭或链接失效
→ 提示订单不可支付
```

## 5. 用户故事

### 5.1 销售视角

#### 用户故事 1

作为销售人员，我希望在订单列表中直接发起代客户下单，这样我可以在和客户沟通时快速帮客户完成下单动作。

#### 用户故事 2

作为销售人员，我希望先选会员，再自动带出该客户默认收货地址，这样我无需重复录入地址信息。

#### 用户故事 3

作为销售人员，我希望可以搜索微商城商品、调整数量并删除商品，这样我能像帮客户搭配购物车一样完成订单。

#### 用户故事 4

作为销售人员，我希望默认邮费为 `0` 且可手动修改，这样我能根据实际配送情况灵活处理邮费。

#### 用户故事 5

作为销售人员，我希望在必要时可以直接修改订单总价，这样我能在沟通成交时快速完成议价处理。

#### 用户故事 6

作为销售人员，我希望提交成功后系统自动复制付款链接并返回订单列表，这样我可以立刻把付款链接发给客户。

#### 用户故事 7

作为销售人员，我希望待付款的代客户下单支持编辑、删单、复制付款链接，这样我在客户尚未付款前仍可继续跟进和调整。

### 5.2 客户视角

#### 用户故事 8

作为客户，我希望点击销售发给我的付款链接后能直接进入订单详情页并完成付款，这样我不用重新挑选商品。

#### 用户故事 9

作为客户，我希望也能在自己的订单中心待付款列表里看到这笔订单，这样即使没及时打开链接，也能从小程序继续支付。

1. 验证未选择会员时不可提交
2. 验证未选择商品时不可提交
3. 验证未填写地址时不可提交
4. 验证库存不足时不可提交
5. 验证订单金额计算结果与页面展示一致
6. 验证手动修改订单总价后，客户支付金额以最新金额为准
7. 验证订单编辑后，客户侧订单详情展示最新内容和最新金额
8. 验证删单后原付款链接不可再支付

### 6.3 异常测试

1. 验证无权限账号不可见或不可使用代客户下单入口
2. 验证会员不存在默认地址时，系统要求销售补充地址
3. 验证剪贴板复制失败时，系统给予明确提示但不影响订单创建成功
4. 验证支付链接过期或订单已关闭时，客户无法继续支付
5. 验证网络异常场景下，提交失败不会生成重复订单
