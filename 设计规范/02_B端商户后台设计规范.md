# B端商户后台设计规范

## 1. 设计目标

本规范用于指导 OpenDesign 生成「商家自营售后体系」B端商户后台原型。

B端后台重点表达：

1. 商家在后台查看售后订单列表。
2. 商家筛选待审核、待寄回、待收货、待退款、退款中、退款失败、待补寄出库、待买家收货、已完成、已关闭等售后单。
3. 商家进入售后详情，查看原订单信息、退款信息、退货信息、申请信息。
4. 商家根据售后状态执行操作：同意申请、拒绝申请、转补寄处理、签收质检、操作退款、重新操作退款。
5. 售后流程完成后，后台状态同步更新。
6. B端页面必须是商户后台风格，不要做成 C端手机页面。

---

## 2. 整体视觉风格

B端商户后台整体应呈现：

- 信息密度高；
- 表格化；
- 白底 / 浅灰背景；
- 蓝色作为主操作色；
- 红色用于金额、异常、退款、驳回提示；
- 灰色用于已完成、不可操作、辅助信息；
- 卡片分区清晰；
- 弹窗承载复杂操作。

页面应像真实 SaaS 管理后台，而不是小程序页面。

## 8. 筛选区规范

售后订单列表顶部应包含筛选区。

当页面目标是「商户后台售后管理列表」时，筛选区优先参考以下结构，不要扩展成多层胶囊筛选工作台：

```text
左侧：时间筛选标签 + 时间类型下拉（如：创建时间） + 日期范围
右侧：关键词搜索框 + 查询按钮
```

该模式用于快速处理售后单，同时要保留 PRD 要求的核心筛选字段。应优先保证：

- 首屏一眼能看到状态分布和表格内容；
- 筛选层数尽量少；
- 高级筛选只保留 PRD 明确要求的字段，不自行扩展额外后台概念；
- 搜索框默认支持：售后单号 / 订单号 / 手机号。

常见筛选项：

```text
时间筛选：创建时间 / 申请时间 / 审核时间
日期范围
售后状态
关闭类型
售后类型
订单业务形态
订单来源 / 营销类型
交易门店
分账状态
责任方
搜索框（售后单号 / 订单号 / 手机号）
查询按钮
```

样式：

```text
筛选区背景：#FFFFFF
控件高度：40px - 48px
输入框边框：#D9D9D9
输入框圆角：8px
按钮圆角：8px
控件间距：8px - 12px
```

搜索框提示：

```text
请输入原订单号 / 退款申请人姓名 / 手机号搜索
请输入直播名、会员手机号、订单号、支付单号搜索
```

## 9. 表格规范

### 9.1 售后订单列表表格

当采用「参考图版式」时，状态区与表格字段应优先按以下结构组织：

```text
状态条：
待审核 / 待寄回 / 待收货 / 待退款 / 退款中 / 退款失败 / 待补寄出库 / 待买家收货 / 已完成 / 已关闭 / 全部

表格字段：
售后单号
类型
商品信息
用户
门店
售后原因
运费责任
退回物流
退款金额
状态
操作
```

其中：

- 状态区放在筛选区下方、表格上方，使用一整排横向状态切换；
- 每个状态项应包含图标、文字、数量；
- 当前状态使用蓝色文字 + 蓝色底数量徽标 + 底部蓝色线条；
- 表格表头使用浅灰底分栏，形成强表格感；
- 操作列只保留售后详情和当前节点主操作，不要在列表区堆叠解释性文案。

建议字段：

```text
申请时间
申请人
退款类型
退款商品
申请退款金额
退款申请信息
原订单信息
所属门店
售后状态
操作
```

字段说明：

- 申请时间：日期和时间可换行展示。
- 申请人：展示姓名和手机号。
- 退款类型：仅退款 / 退款退货。
- 退款商品：展示商品名称，过长可省略。
- 申请退款金额：展示 ¥ 金额。
- 退款申请信息：展示申请类型、联系电话、申请原因、申请说明、退货单号、退货留言、操作人、操作时间等。
- 原订单信息：展示订单编号、订单类型、订单状态、物流单号、订单金额、下单商品。
- 所属门店：展示门店名称。
- 售后状态：展示待审核、待寄回、待收货、待退款、退款中、退款失败、待补寄出库、待买家收货、已完成、已关闭。
- 操作：根据状态展示售后详情及当前节点主操作；例如同意申请、拒绝申请、转补寄处理、签收质检、操作退款、重新操作退款。

### 9.2 表格样式

```text
边框：#E8E8E8
```


## 19. OpenDesign 生成约束

生成 B端商户后台页面时，请严格遵守：

1. B端必须是后台管理系统风格，不要做成手机页面。
2. 页面应包含左侧导航、顶部业务 Tab、筛选区、表格区。
3. 售后订单列表必须以表格为主。
4. 售后详情应以弹窗或抽屉形式展示。
5. 详情弹窗内必须分区展示：售后进度、原订单信息、退款信息 / 退货信息、售后操作信息。
6. 商家操作必须通过弹窗确认，不要直接完成状态。
7. 操作按钮应放在详情弹窗底部固定操作栏。
8. 主操作按钮使用蓝色。
9. 金额、退款、驳回、异常提示使用红色。
10. 风险提示和重要说明使用橙色。
11. 不要把 C端消费者页面混入 B端页面。
12. 不要做三栏联动看板。
13. 一个业务状态流程应独立成一个流程面板。
14. B端流程面板中每个节点应是后台页面、详情弹窗或操作弹窗。
15. 已完成状态不应展示可操作按钮。
16. 待处理状态必须展示对应操作入口。
